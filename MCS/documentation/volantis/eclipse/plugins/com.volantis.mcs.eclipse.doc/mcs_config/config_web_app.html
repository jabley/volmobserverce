web-application

Purpose

Configures MCS with a web application server.

Note: The deprecated attributes will be removed from this reference in a later release. You should use the jdbc-repository or named-datasource elements to define datasources.

Contained by

mcs-config

Attributes

Name Description Type Default Options Use
anonymous If it is possible to access the connection pool anonymously, and no user name or password is required, this attribute can be set to true. The user and password attributes will then be ignored and the connection retrieved anonymously. xs:token false true, false deprecated
app-server-name Identifies an application server interface for connection pool handling xs:string none Dynamo511, Tomcat31, Weblogic60, Websphere40, Websphere50 optional
base-url Defines the effective URL of the server seen from a client. Used only when there is some URL mapping in use. xs:anyURI none optional
datasource The name of a data source. If a the connection pool attribute is used this must be set. xs:string none deprecated
datasource-vendor The name of a data source vendor. If a the connection pool attribute is used this must be set. xs:string none deprecated
internal-url Defines the absolute URL of the server as seen from the web application. Used only when there is no other more specific prefix, for example, from associated asset groups, for the absolute part of their URL. xs:anyURI none optional
jndi-provider The location of a JNDI provider. If a BEA Weblogic connection pool is used this attribute must be set. xs:anyURI none deprecated
page-base Used as part of a URL when generating resources. For example, the URL to generated chart images constructed from base-url, page-base, and the chartimages element's base attributes, together with any additional path delimiters. xs:string none optional
password The password. If a the connection pool attribute is used this must be set. xs:string none deprecated
use-server-connection-pool Controls whether the application server's connection pool should be used, if available. If the connection pool is used, the datasource-vendor, datasource, user and password attributes need to be set. xs:boolean false deprecated
user The user name. If a the connection pool attribute is used this must be set. xs:string none deprecated
xml-reader The class name of an XML reader class xs:string none deprecated

Example

<web-application
    base-url="/"
    internal-url="http://localhost:8080/"
    page-base="[context_root]"
    app-server-name="[server_name]"/>
