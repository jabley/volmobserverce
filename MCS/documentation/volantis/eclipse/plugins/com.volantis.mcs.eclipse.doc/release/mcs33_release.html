MCS 3.3 release notes

The following is a summary of the changes to features and functionality in MCS 3.3. Further information and details about existing functionality and features may be found in previous release notes and documentation.

You should also refer to the notes in Supported platforms and Application server issues.

Java and Eclipse

From version 3.3 the MCS policy editors require JDK 1.4.2 and Eclipse 3.0 or later. JDK 1.2 is no longer supported.

Layout styles

In the Layout editor you can now associate themes with the following formats.

  • Grid
  • Grid column
  • Grid row
  • Spatial iterator
  • Spatial iterator column
  • Spatial iterator row

Drag and drop

Drag and drop support has been added to the Eclipse environment for MCS. This support is mainly for layouts, but it is also now possible to drag image files into the assets section of an image component editor to create assets for that component.

There is an issue on the Linux version of Eclipse which will be fixed with the 3.1 release of Eclipse. If a layout format is dragged and dropped onto an empty format as a copy on the Design page, Eclipse will hang. This is not a problem in the Outline view.

Spatial iterators

The use of spatial iterators is extended to include two new contexts.

You can now place iterators inside a form to display tabular data.

It is also possible to enclose regions, so that variable numbers of portlets or content sources can be combined in a single layout. The syntax for using a fixed region name outside a format iterator is unchanged. Inside an iterator, you use an MCS expression layout:getRegionInstance() that takes a string and a series of integer index values to refer to the instances of the regions within the iterator. For example:

<region name="{layout:getRegionInstance('name', 0, 1, 2, 3)}"> ... <region>

Disabling branding

A new method of disabling branding has been implemented.

The deprecated use of the caret (^) character is replaced with the function branding:getUnbrandedPolicy('policyName'), for example 

<img src="^logo.mimg" alt="Logo"/>

should be replaced with the expression

<img src="branding:getUnbrandedPolicy('logo.mimg')"/>

Importing theme information

MCS now supports the importing of style properties in themes from a parent device theme in the device hierarchy. So you can build up complex themes that share core branding styles at say the master level, and adjust these at device family or specific device level. The default is not to import themes. Refer to Adding a theme to see how designers can control theme import.

Numeric menu labels

You can now specify that accesskeys on a menuitem can rendered as numeric labels. Both WML and XHTML devices are supported. The selector pseudo-elements 'shortcut' and 'shortcut-after' take the style rules for the numeric label and a separator.

A new Content category with a single Content property contains the text separating the label from the menuitem. The Menu category has an added Menu Item Shortcut Active control to switch numbering on or off.

White space styling in WML forms

WML styling in XDIME single and multiple select elements is improved in this release.

When you apply a white-space property with a value of 'nowrap' to the xfsiselect or xfmuselect elements in a form, MCS wraps the output in a p element with the mode attribute set to 'nowrap'. If the caption and field are written to the same pane, they are wrapped in the same paragraph, otherwise they are wrapped separately.

WCSS support

This release extends support to the CSS2-based styles defined in WCSS, a subset of CSS2 for WAP2. The WAP-specific styles in this specification are not supported.

MySQL support

Support has been added for the use of the MySQL database for JDBC repositories. Owing to the maximum size of primary key allowed in MySQL, only versions 4.1.6 or later are recommended. There is a compile option that can be set to increase the primary key size on earlier versions, but that may be undesirable.

Device search

The search feature in the Eclipse workbench has been extended to include searching the MCS device repository. The search allows users to search for devices by name or use regular expressions in the search. From the search results, users can navigate to each match in the device editor. See Searching a device repository in for further information.

Null asset

It is now possible to set a 'null' value for assets on every component that can have targetable assets. During fallback, if the first asset encountered for a component is a null asset, then the result is the same as if no asset (of any type) existed for that component anywhere in the fallback path. Refer to Using the asset type attribute for details.

Loading image assets

You can use the load-asset CLI to load large numbers of MCS image components and their assets from the file system or a URL to an XML repository in a single operation. You can rerun the loader at any time following a change one or more images.

You specify a target policy folder and a directory tree containing directories named for each component, and sub-directories for each asset type. (device, convertible or generic). When MCS loads the images, it creates a series of image components in the target policy folder, with assets of the type in the sub folders. Image metadata is also interpreted to generate the necessary attributes automatically.

Changes to MarinerRequestContext API

The method MarinerRequestContext.setContentTypeCharSet(String) has been replaced with MarinerRequestContext.setCharacterEncoding(String).

Changes to MCS documentation

As part of an ongoing transition to modular documentation, the guide to Dynamic Content Integration, and the entire XDIME element reference have been added to the the Eclipse documentation along with MCS Help. The MCS Administration Guide and the related MCS Configuration Element Reference are also available in Eclipse.

See MCS site map for an overview of available documents.
