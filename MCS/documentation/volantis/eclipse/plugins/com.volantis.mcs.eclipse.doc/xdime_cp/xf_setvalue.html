xf:setvalue

Purpose

Associates a specific xf:submit control with the value to be submitted when there are multiple submit buttons on a form.

Contained by

xf:submit

Attributes

AttributeDescriptionTypeDefaultOptionsUse
ev:eventA global attribute defined within the XML Events namespace. The namespace declaration http://www.w3.org/2001/xml-events must be declared. The only value supported is 'DOMActivate'.xs:string DOMAvtivate required 
refIdentifies the request parameterxs:stringnone  optional 
valueThe value to be submitted. If it is not present then the content of the element is used as the value. If the content is empty then it is equivalent to specifying a value attribute as an empty string. The content model for the xf:setvalue element is PCDATA.xs:stringnone  optional 

Attribute groups

Core attributes, Selection attributes, Form attributes

Examples

In the first example, if the 'Reject' button is clicked, the URL will be http://localhost:8080/volantis/response.jsp?on_click=reject

<xf:model>
   <xf:submission id="submit" action="http://localhost:8080/volantis/response.jsp"/>
</xf:model>
   ...
<xf:submit submission="submit">
   <xf:label>Accept</xf:label>
   <xf:setvalue ev:event="DOMActivate" ref="on_click" value="accept"/>
</xf:submit>
<xf:submit submission="submit">
   <xf:label>Reject</xf:label>
   <xf:setvalue ev:event="DOMActivate" ref="on_click" value="reject"/>
</xf:submit>

It is also possible to discriminate on the basis of the ref attribute, in which case the URL will be http://localhost:8080/volantis/response.jsp?reject=

<xf:model>
   <xf:submission id="submit" action="http://localhost:8080/volantis/response.jsp"/>
</xf:model>
   ...
<xf:submit submission="submit">
   <xf:label>Accept</xf:label>
   <xf:setvalue ev:event="DOMActivate" ref="accept" value=""/>
</xf:submit>
<xf:submit submission="submit">
   <xf:label>Reject</xf:label>
   <xf:setvalue ev:event="DOMActivate" ref="reject" value=""/>
</xf:submit>
