Creating a project

A project is a set of resources that you define for a particular web application. Resources that make up a project could include XML or JSP files, assets, layout, theme, component and device policies.

MCS organizes resources according to project, so you can arrange for multiple projects to share the same policy repository.

In MCS, you can use the New Project wizard to add a new project to the Workbench. The wizard allows you to define:

  • A project name and location
  • Project settings for a device repository and a source directory

Naming the project

The first page allows you to locate and name the project, and the location of the project contents.

  1. Choose the MCS Project wizard
  2. Click Next
  3. Enter a name in the Project name text box
  4. To choose a location other than the Workbench, uncheck the Use default checkbox, and click Browse to select a system folder
  5. Click Next

Defining project settings

On the next page you must enter values for the location of the project device repository, and name the source directory. There can only one policy source directory per project, and all your MCS resources must be in this directory or one under it.

The device repository can be any one on your system that is appropriate for this project. The default path of the source directory is 'WebContent/mcs-policies', but you can use any name you wish.

  1. Use the Browse button to locate the device repository provided with MCS. The repository filename must have a .mdpr extension.
  2. Enter a name in the Policy source directory control, or accept the default
  3. Click Finish. The new project icon appears in the Navigator pane
  4. To view the project contents, select the icon and choose File | Refresh. Now expand the project.

Adding MCS resources to an existing project

You can add MCS resources to an existing project, such as a company web site, by adding a folder to your project. This will be your policy source directory. You do not have to create an MCS project.

  1. Choose an existing folder or right click on a project folder and choose New | Folder. With a new folder you can use any name, but a name like 'MCS policies' will be meaningful.
  2. Now choose an MCS wizard for the resource you want to add
  3. On the MCS settings page in the wizard click Browse to locate a device repository for the project
  4. Continue with the wizard to complete adding the resource

MCS displays the resource icon in the new folder, and opens an editor on the resource.
