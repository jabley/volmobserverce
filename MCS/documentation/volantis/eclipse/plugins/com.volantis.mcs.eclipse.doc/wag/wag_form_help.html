Form help and prompts

You can supply help and prompts to help users complete XDIME forms, at both form and individual control level. Nearly all form elements have a help and a prompt attribute, or both. The attributes can contain literal text, a MCS or JSP expression, or VoiceXML that is valid in the context of the element in use.

Note: MCS does not generate any VoiceXML markup. The entire content to be processed by the browser must be contained in the attributes.

Help

Help is not supported in either HTML or WML. In VoiceXML, MCS renders the content for the help element. The example repeats the most recent prompt after a short delay.

<help>Select the drink you would like.
   <break msecs='500'/>
   <reprompt/>
</help>

Prompts

Prompts are not supported in WML. In HTML 4.0 or later, MCS uses the prompt content to display pop-up help - earlier versions do not support help. In VoiceXML, MCS renders the content for the prompt element.

<prompt>Would you like coffee, tea or milk?
</prompt>

You have complete control over the VoiceXML used in prompts, so it is possible to use the full range of language features, including counts. For example, you can provide different prompts depending on the number of times the prompt has been delivered.

<prompt>Enter the item number.</prompt>
<prompt count="3">Say the item number of the product you
wish to buy, or enter it on the keypad.
</prompt>
<prompt count="5">If you need help, enter zero.
Otherwise, say the item number of the
product you wish to buy, or enter it on the keypad.  
</prompt>

With complex, multiple prompts to process, you may find it better to code the attribute as a MCS expression.

You can also use audio prompts in place of text.

<prompt>
   <audio src="drinks.wav">Would you like coffee, tea or milk?</audio>
</prompt>

In this version, the audio clip drinks.wav is played if available. If not, the text in the body of the audio tag is played instead.

Note: When you reference external assets from a prompt, the reference is to the asset itself, and not to an MCS component. To provide different assets for different devices, you can create targeted text variants to contain the different prompts.
