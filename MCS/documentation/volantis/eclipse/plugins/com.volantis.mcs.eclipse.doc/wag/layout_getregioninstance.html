layout:getRegionInstance

Purpose

Returns a reference to one instance of a region inside an iterating format. The reference can be used wherever a region name can be used.

Note: The special type layout:regionReference may be used only with functions that return it for use in attributes that reference regions.

Signature

layout:getRegionInstance($regionStem as xs:string, $index as xs:integer, ...) as layout:regionReference

Parameters

NameDescriptionType
index

The index of the instance of the region required. There must be at least one of these values. There may be more if the reference requires it.

xs:integer
regionStem

The name of the region as it appears in the layout without any index dependent information

xs:string

Example

<region
   name="{layout:getRegionInstance('name', 0, 1, 2, 3)}">
   ...
<region>
