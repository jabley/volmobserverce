Directing web servers

You can use server side includes (SSI) in XDIME to direct a web server to do some basic processing. Some site configurations include web servers in addition to application servers. They deliver static content, and can supply some additional processing, which is controlled by directives embedded in the response to a request.

For example, you can include content directly in the generated response without requiring the browser to request it independently. Individual content components can be cached when originally generated, and then replaced in the response from the application server by SSI directives that cause their inclusion directly from the cache.

Note: The included material is not processed in any way by the web server. The XDIME inclusion mechanism enables support for caching systems, but is not intended as a replacement for other types of inclusion.

The SSI directives

You use the ssiconfig to included additional material.Two mutually exclusive attributes specify the added content. The virtual attribute defines a URL on the web server. This form of include is useful when the material is remote from the directory containing the page with the directive. The file attribute defines a relative file path. It is probably most useful when the include is in the same directory that contains the page, or is close to it.

The ssiconfig element can contain an error message to be written in the event of a failure. It takes a single errmsg attribute.

<wsdirectives>
   <ssiconfig errmsg="The file 'Q4report.html'is not available."/>
   <ssiinclude virtual="/mycorp/sales 2004/Q4report.html"/>
</wsdirectives>
