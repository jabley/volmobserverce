Enabling MIME packaging

You can enable MIME packaging features in the page-packaging section of the mcs-config.xml. Many WAP and XHTML devices support delivery of pages and their variants using multipart MIME as a container. To improve performance, MCS can package content in the form of multipart MIME documents, delivering the requested page along with the variants that it references.

To enable MIME packaging you should set the enabled attribute of the mime-packaging to 'true'.

<page-packaging>
   <mime-packaging enabled="false"/>
</page-packaging>

When a request can be satisfied with a multi-part MIME package, the HTTP accept header will indicate that the requester can accept a response with content type multipart/mixed or application/vnd.wap.multipart.mixed.

The Protocol category in the MCS Device Repository editor contains a setting for MIME packaging. The Use MIME packaging dropdown allows you to set three possible values:

  • default - MCS will use packaging if the accept header supports multipart MIME
  • always - MCS will use packaging regardless of the header content
  • never - MCS will never use packaging even if the header indicates support
Note: To use multipart MIME you must install the JavaMail classes (JavaMail 1.3.2) and the JavaBeans Activation Framework (JAF 1.0.2). Both packages are available from http://java.sun.com/.
