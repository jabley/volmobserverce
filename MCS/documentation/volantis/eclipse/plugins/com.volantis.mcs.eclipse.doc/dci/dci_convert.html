Converting content

If you need to convert pipeline content, MCS provides conversion elements for:

  • Text
  • Images
  • URLs

Converting text

If you need to directly translate to a native markup that only supports a certain case, such as WML, you can use the pipeline:convertElementCase element. It switches the case of all element names in its input to either lowercase or uppercase and passes this to the output. You use the mode to specify the case value.

<pipeline:convertElementCase mode="lower">
   <urid:fetch href="http://www.mysite.com/index.html" parse="xml" />
</pipeline:convertElementCase>

Note: This type of conversion is distinct from any conditioning of input markup by the DCI web driver. See Defining HTTP resources for details.

Converting images

You can use the pipeline:convertImageURLToTranscoder to rewrite convertible image URLs in a page to be routed through the Image Conversion Service (ICS). ICS will use the device characteristics to select the best transcoding rule to use. The required server attribute contains the address of the ICS server. You can also control the file size and the scaling of the output image with maxfilesize and relativeWidth attributes.

Refer to About ICS for details of ICS conversion features.

<pipeline:convertImageURLToTranscoder 
   server="http://ics.mysite.com:8080" 
   maxfilesize="9000"
   relativeWidth="65">
   <urid:fetch href="http://www.mysite.com/index.html" parse="xml" />
</pipeline:convertImageURLToTranscoder>

Converting URLs

To translate URLs in content, or strip a specified prefix, you can use pipeline:convertAbsoluteToRelativeURL. Used alone, the baseURL attribute removes the specified part of the path from the input. If you also provide a value in the substitutionPath, MCS will replace the value in baseURL. The output in the exmple would be /samples/xml/index.html.

<pipeline:convertAbsoluteToRelativeURL
   baseURL="http://www.mysite.com/"
   substitutionPath="/examples/xml/"/>
   <urid:fetch href="http://www.mysite.com/index.html" parse="xml" />
</pipeline:convertAbsoluteToRelativeURL>
