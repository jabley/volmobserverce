Effects properties

Purpose

Defines style properties used by the Framework Client.

ControlDescriptionOptions
Marquee styleThe effect that will be used for the marqueenone (default), slide, scroll, alternate 
Marquee directionThe initial direction in which the marquee content will moveleft (default), right 
Marquee repetitionThe number of times that the content will move across the screen. The default value is 1. 
Marquee velocitySpecifies the velocity that the marquee content will move across the screen. A negative velocity value will reverse the direction of movement.normal, length, length/time 
Frame rateSpecifies the frequency in Hz that is required when displaying visual effects such as transitions, or scrolling. The default value is 10Hz. The device policy value x-client.frame.rate.max sets an overriding upper limit. 
Effect durationThe duration in seconds of the effect applied to the widget element. The default value is 1.time (default value is 1s) 
Effect styleControls the transition effectsnone (default), fade, slide-left, slide-right, slide-top, slide-bottom, wipe-top, dropout, grow-center, grow-top-left, grow-top-right, grow-bottom-left, grow-bottom-right, shrink-center, shrink-top-left, shrink-top-right, shrink-bottom-left, shrink-bottom-right, puff, shake, pulsate, highlight, random 
Transition intervalThe duration in seconds of the content replacement in a Carousel widget. The default value is 1. 
Widget initial stateSpecifies the initial state of the widgetinactive (default), active 
Toggle eventDescribes the event on which a submenu will be unfolded. Used by the Folding Item, Field Expander, and Dynamic Menu widgets.click (default), focus  
Validation error actionControls actions that will occur after a validation failure. Used by the Simple and Multiple Forms Validator widget. Applies to the input element.none (default), message, focus 
CancelableEach wizard step may optionally include this 'Cancel' action. Applies to xf:group and widget:wizard.always (default), never 
Rounded cornersAllows boxes to be displayed with rounded corners. The first value defines horizontal radius of the selected corner, while the second specifies its vertical radius. When the second value is not specified, then it is assumed to be equal to the first one and the corner is thus a quarter circle. If either value is zero then the corner is squared. Negative values are not allowed.
