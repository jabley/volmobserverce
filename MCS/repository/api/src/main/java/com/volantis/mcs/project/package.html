Provides interfaces and classes for representing and managing projects within MCS.

See compatibility for information about how the classes in this and sub packages relate back to the old component / asset objects.

Overview

A {@link com.volantis.mcs.project.Project project} is an single entity that encompasses all the information needed by MCS that is not specific to an instance of MCS, i.e.

  • Project specific configuration.
  • Policies.
  • Pages.
  • Resources.

At the moment the main focus for projects are the policies but this will expand in future to include the other aspects.

{@link com.volantis.mcs.project.Project} instances can be constructed using the {@link com.volantis.mcs.project.ProjectFactory}.

Compatibility

In addition to some new functionality this package contains replacements for the following now deprecated classes:

  • {@link com.volantis.mcs.repository.RepositoryManager}
  • {@link com.volantis.mcs.repository.AssetGroupRepositoryManager}
  • {@link com.volantis.mcs.repository.AudioRepositoryManager}
  • {@link com.volantis.mcs.repository.ChartRepositoryManager}
  • {@link com.volantis.mcs.repository.DynamicVisualRepositoryManager}
  • {@link com.volantis.mcs.repository.ImageRepositoryManager}
  • {@link com.volantis.mcs.repository.LayoutRepositoryManager}
  • {@link com.volantis.mcs.repository.LinkRepositoryManager}
  • {@link com.volantis.mcs.repository.ScriptRepositoryManager}
  • {@link com.volantis.mcs.repository.TextRepositoryManager}

All of the classes above that extend {@link com.volantis.mcs.repository.RepositoryManager} have been replaced by a single class {@com.volantis.mcs.project.PolicyManager}. It has many advantages over the previous classes:

  • It is a single consistent interface to managing policies. The ...RepositoryManager classes each have their own special interfaces for managing different component and policy types.
  • It frees the user code from having to manage {@link com.volantis.mcs.repository.RepositoryConnection RepositoryConnections}, in particular it does not have to clean them up.
  • The {@link com.volantis.mcs.project.BatchOperation} also means that user code does not have to do any clean up when performing a number of operations together, although it can if necessary.

Previously, the {@link com.volantis.mcs.repository.Repository} was the main focus for managing policies but this has gradually been changing to the point where now the {@link com.volantis.mcs.project.Project} is the focus and the {@link com.volantis.mcs.repository.Repository} is a supporting class for the project.
