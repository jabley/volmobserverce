Calling custom classes

If you need to call specific code in external applications from XDIME content, you can invoke a custom class with elements from the vtim namespace and pass any necessary parameters to it.

These custom classes, also known as markup plugins, can be specified in the markup-plugins element in the mcs-config.xml file. Refer to Defining markup plugins for configuration details.

You use the vtim:invoke element to call methods on these classes from XDIME markup. The pluginName attribute contains the name the plugin defined in the configuration file, and methodName contains the name of the method to be invoked. Both attributes are required.

Possible values for methodName attribute correspond to the methods specified in the class interface definition.

  • initialize performs any necessary initialization on the class. Note that it is possible to cause initialization to occur before pages are processed using facilities in the XDIME configuration.
  • process processes any data
  • release is called automatically by MCS when the class is about to be removed. You can also call release explicitly.

The vtim:arguments element contains an vtim:argument element for each argument to the method called. The paired name and value attributes match the arguments specified in the equivalent element in the configuration file.
