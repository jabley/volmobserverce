pipeline:cacheKey

Purpose

Specifies part of the composite key attribute, when there are multiple keys in use. The key is added to the composite key maintained by the caching process. Values may be literal text values, or more complex expressions. The order of the individual keys making up the composite key is significant.

Contained by

pipeline:cacheInfo

Attribute

AttributeDescriptionTypeDefaultOptionsUse
valueSpecifies a partial key used to access the cachexs:stringnone  required 

Example

In the example the keys refer to values passed to template functions.

 <pipeline:cacheInfo>
   <pipeline:cacheKey value="%{request:getParameter('id')}"/>
   <pipeline:cacheKey value="%{device:getDevicePolicy('custom.policy')}"/>
</pipeline:cacheInfo>
