Working with XDIME

XDIME (XML Device Independent Markup Extensions) is an XML vocabulary with elements that partly reflect the usage of tags in languages such as HTML and WML, though the exact syntax and attributes may not be identical. This makes it very easy to create XDIME content if you already know these languages. These are elements for block and inline text, hyperlinks, and inserting images and other objects.

Additionally there are several groups of elements that are specific to XDIME.

  • A flexible container structure that allows for nesting and the maintenance of reuseable code fragments
  • Binding to components, layouts and themes
  • Page caching options
  • Tags for forms using XForms concepts and allowing form fragmentation
  • Elements for menu structures
  • Conditional markup
  • An extensive vocabulary for dynamic content integration
  • Messaging elements

The example contains a canvas, or page, that is bound to a layout in the layoutName. The logo defines an image component. The layout has three panes. Two panes are defined with pane elements, and a third pane defined with the alternate pane attribute syntax on the paragraph.

<canvas
   layoutName="/example.mlyt"
   theme="/example.mthm">
   <logo pane="logo1" src="/stars.mimg"/>
   <pane name="cpynews">
      <h2>Floods in Britain set to worsen</h2>
      <p>Weather forecasters are predicting further strong winds and
      heavy rain this weekend, adding to the misery for the
      thousandsof people whose homes have been flooded during the
      last two weeks. With rivers at record levels, the worst floods
      for over 50 years have forced large numbers of people to seek
      shelter elsewhere.</p>
   </pane>
   <pane name="headlines">
      <h2>Other Stories</h2>
         <p><a href="elvis.xml">Nun sees Elvis on moon</a></p>
         <p><a href="man.xml">Man bites dog</a></p>
         <p><a href="hamster.xml">Hamster abducted by aliens</a></p>
   </pane>  
   <p pane="trailer"><i>Copyright 2004 NewsService Group</i></p>
</canvas>
