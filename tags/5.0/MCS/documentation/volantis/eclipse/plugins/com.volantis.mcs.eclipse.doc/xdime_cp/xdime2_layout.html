XDIME 2 and layouts

In XDIME 2, the html element is the root container, and attributes on the link tag bind the page to a layout or theme. If these attributes are not present, MCS will use the default project values for layout and theme, and failing that will attempt to render a page using built in values.

<head xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs">
   <title>Welcome</title>
   <link rel="mcs:layout" href="/mylayout.mlyt"/>
   <link rel="mcs:theme"  href="/mytheme1.mthm"/>
   <link rel="mcs:theme"  href="/mytheme2.mthm"/>
</head>

XDIME 2 markup defines presentation with theme properties, two of which control layout.

  • The Target Layout property names the layout policy to be used
  • An element can be directed to an area within the current layout Target Container.

You can also use CSS expressions with iterator panes.

Target layout

When you assign the Target Layout property to an element it causes that element and all its contents to be targeted to that layout. For example, you might specify a class 'itemlist' for a navigation list and set the property to '/mylayout.mlyt', and use the following markup.

<nl class=”itemlist”>
   <li class=”item”>
      <object src=”image1.png” srctype=”image/png”/>
      <a href=”link1.xdime”>Link 1</a>
      <div class=”label”>Label 1</div>
   </li>
   ...
</nl>

Target container

Because a layout can have several potential containers, you must also specify the one in which the XDIME 2 content should be placed. You do this using the Target Container property. It is an error if an element with a Target Layout property does not also have Target Container setting. However, you can use different rules for the two properties if required.

Container references

A container reference can either be the name of a container represented as a CSS string, or a CSS expression that uses the mcs-container-instance function to select a specific instance of a container to use. This is equivalent to the layout:getPaneInstance() and layout:getRegionInstance() function expressions that you can use in XDIME.

However, this is not recommended, because in most cases it is not possible to predict how many instances there will be. Instead you should to use the counter function.

counter($name as xs:string) as xs:integer

Unlike a CSS counter, the name of the counter is specified as a string, rather than a CSS identifier, and the optional style parameter is not needed. The allowable values are however identical.

<html xmlns="http://www.w3.org/2002/06/xhtml2"
   xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs">

   <head>
      <title>Iterated Page</title>
      <link rel="mcs:layout" href="/IteratorPage.mlyt"/>
      <link rel="mcs:theme" href="/IteratorPage.mthm"/>
      <!-- The items and blocks counters are reset to -1. -->
      <style type="text/css">
         body {counter-reset: items -1 blocks -1}
         div.item {counter-increment: items; mcs-container: mcs-container-instance('item', counter(items))}
         div.block {counter-increment: blocks; mcs-container: mcs-container-instance('block', counter(blocks))}
      </style>
   </head>
   <body>  
      <!-- items counter incremented to 0, content targeted at pane item.0 -->
      <div class="item">Pane 0</div>
      <!-- items counter incremented to 1, content targeted at pane item.1 -->
      <div class="item">Pane 1</div>
      <!-- items counter incremented to 2, content targeted at pane item.2 -->
      <div class="item">Pane 2</div>

      <!-- wrapper equivalent to an XDIME region element
           blocks counter incremented to 0, content targeted at region block.0 -->
      <div class="block">
          <!-- wrapper equivalent to an XDIME canvas element -->
          <div style="mcs-layout: '/InnerPage.mlyt'">
                <p style="mcs-container: 'Pane'">Some inner content 0</p>
          </div>
      </div>

      <!-- another 'region'
           blocks counter incremented to 1, content targeted at region block.1 -->
      <div class="block">X
          <!-- another 'canvas -->
         <div style="mcs-layout: '/InnerPage.mlyt'">
            <p style="mcs-container: 'Pane'">Some inner content 1</p>
         </div>
      </div>
   </body>
</html>

The scope of a container name is that of the innermost containing layout; normally be the layout of the block element containing the element that references the container. The block element or any elements that it contains may reference containers within the layout.

If the pane does not exist within this scope, the output of that content is suppressed. If you do not specify a container, the content is directed to the area where the parent element's content is written.

You can use a number of optional XDIME elements to define or override the link texts on fragments at run time. These are values that you might have omitted in the Layout editor because the layout is not always fragmented, or that you set as defaults to be changed in specific contexts.

Overriding fragment links

In your XDIME 2 content, you can override fragment and form fragment link texts at run time, using the meta and xf:group elements. These are values that you might have omitted in the Layout editor because the layout is not always fragmented, or that you set as defaults to be changed in specific contexts. See Fragment links in XDIME 2 and Form fragments links in XDIME 2 for details.
