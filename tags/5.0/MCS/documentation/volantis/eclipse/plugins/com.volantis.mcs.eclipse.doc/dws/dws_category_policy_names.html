category-policy-names

Purpose

Lists the names of all the device policies in a category.

If the category name is missing or invalid, '404 Not Found' is returned. If the repository contains no categories, a document with an empty policy-names element is returned.

Method

GET

Signature

/category/policy/names/[category]

Parameter

NameDescriptionType
category

The category to list

xs:string

Examples

Input

/category/names/protocol

Returns

<policy-names xmlns="http://www.volantis.com/xmlns/2006/01/device-ws" category-name="protocol">
    <policy-name>protocol.content.type</policy-name>
    <policy-name>protocol.css.properties.font-size.ex.scaling.factor</policy-name>
    <policy-name>protocol.css.properties.font-size.medium.size</policy-name>
    <policy-name>protocol.css.properties.font-size.medium.size.units</policy-name>
    <policy-name>protocol.mime.fully.qualified.urls</policy-name>
    <policy-name>protocol.mime.preserve.urls</policy-name>
    <policy-name>protocol.mime.urls.to.cache</policy-name>
    <policy-name>protocol.stylesheet.location.device</policy-name>
</policy-names>
