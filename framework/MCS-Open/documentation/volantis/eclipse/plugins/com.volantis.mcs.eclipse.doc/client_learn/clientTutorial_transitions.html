Applying effects

Some widgets support transitions, such as sliding, wiping, and folding. Effects are used when widgets appear on the screen, or disappear. There are also number of effects that can be used to draw user attention to a widget that is not in transition.

About transitions 

The Effect style property allows to applies transition effect to the widget element. In the following example the grow-center effect will be used.

<style type="text/css" media="handheld, all">
  widget|block {
    mcs-effect-style: grow-center
  }
  widget|block:mcs-concealed {
    mcs-effect-style: shrink-center
  }
</style>

The pseudo-class :mcs-concealed set on the widget element defines a disappearing transition. In our case, the shrink-center effect will be used to hide the widget.

By modifying values of mcs-effect-style and mcs-concealed {mcs-effect-style: } you can change the appearing and disappearing transition effects.

Transition effects 

The table lists effects which are useful when a widget itself or its element appears or disappears.

Effect style  Description  Options 
none  No effect is used, the box simply appears or disappears immediately  default 
fade  Box fades in when appearing and fades out when disappearing   
slide-left, slide-right, slide-top, slide-bottom  A slide effect to or from a specific edge   
wipe-top  A wipe effect to or from a specific edge   
dropout  Box drops and fades out   
fold-top-left  Box folds to the specified corner   
grow-center, grow-top-left, grow-top-right, grow-bottom-left, grow-bottom-right  Box grows either from center or from one of the corners   
shrink-center, shrink-top-left, shrink-top-right, shrink-bottom-left, shrink-bottom-right  Box shrinks either to center or to one of the corners   
puff  Box expands and fades away   

Other effects, which indicate to the user that an widget has changed in some way, include shake, pulsate and highlight.
