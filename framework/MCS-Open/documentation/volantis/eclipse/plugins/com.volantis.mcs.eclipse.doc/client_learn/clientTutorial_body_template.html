Content integration

The XDIME 2 pages used by the Client Framework sample application have similar structure. All of them contain header, body and the footer sections. It is possible to define those common elements in a single file and then refer to them from various of pages.

Template elements use the template: prefix. They define common elements which will be used to render the bound XDIME 2 page.

Defining templatesÂ 

  1. Create a demo-main.xdtpl file in the templates directory
  2. Your template starts with the template:definition element. It must contain the namespace declarations including one for the template prefix.
    <?xml version="1.0" encoding="UTF-8"?>
<template:definition xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs"
  xmlns:template="http://www.volantis.com/xmlns/marlin-template"
  xmlns:urid="http://www.volantis.com/xmlns/marlin-uri-driver">
  ...
</template:definition>
  3. It takes two parameters using the template:parameter element. The first one defines the name of the widget. The 'simple' type of a parameter allows character data only. Second parameter defines widget content. The 'complex' type means it may contain character data and XDIME markup. These parameter values are referenced in the template:value elements in the body.
    <template:declarations>
  <template:parameter name="title" type="simple"/>
  <template:parameter name="content" type="complex"/>
</template:declarations>
  4. The template:body element defines the structure of the XDIME 2 page. Add div elements to specify the main sections, using the 'mcs-container' style attribute values to identify the layout container for the content in each case.
    <template:body>
  <div id="logo" style="mcs-container: 'logo'"/>
  <div id="heading" style="mcs-container: 'heading'">
    <object src="/images/heading.mimg"/>
  </div>
  <div id="title" style="mcs-container: 'title'">
    <template:value ref="title"/>
  </div>
  <div id="content" style="mcs-container: 'content';">
    <template:value ref="content"/>
  </div>
  <div id="footer" style="mcs-container: 'footer'">
    <ul>
      <li>
        <a href="index.xdime">Site homepage</a>
      </li>
      <li>
        <a href="http://www.volantis.com">Volantis homepage</a>
      </li>
    </ul>
  </div>
  <div id="copyright" style="mcs-container: 'copyright'">
    Copyright Volantis 2007 </div>
</template:body>

Complete template code

Here is the complete code for the template demo-main.xdtpl.

<?xml version="1.0" encoding="UTF-8"?>
<template:definition xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs"
  xmlns:template="http://www.volantis.com/xmlns/marlin-template"
  xmlns:urid="http://www.volantis.com/xmlns/marlin-uri-driver">
  <template:documentation> </template:documentation>
  <template:declarations>
    <template:parameter name="title" type="simple"/>
    <template:parameter name="content" type="complex"/>
  </template:declarations>
  <template:body>
    <div id="logo" style="mcs-container: 'logo'"/>
    <div id="heading" style="mcs-container: 'heading'">
      <object src="/images/heading.mimg"/>
    </div>
    <div id="title" style="mcs-container: 'title'">
      <template:value ref="title"/>
    </div>
    <div id="content" style="mcs-container: 'content';">
      <template:value ref="content"/>
    </div>
    <div id="footer" style="mcs-container: 'footer'">
      <ul>
        <li>
          <a href="index.xdime">Site homepage</a>
        </li>
        <li>
          <a href="http://www.volantis.com">Volantis homepage</a>
        </li>
      </ul>
    </div>
    <div id="copyright" style="mcs-container: 'copyright'">
      Copyright Volantis 2007</div>
  </template:body>
</template:definition>

In the next section you will see how the template:binding element is used to associate this template with a specific page.

Checklist

NamePurpose
a

Used to create an explicit link to another place in the same document or to another document. The current document is the source of the link and the value of the href attribute, defines the link target

div

A section used to add extra structure to documents. Style sheets can be used to control the presentation.

li

A list item in an ordered, unordered or navigation list.

object

A reference to a media resource. Object allows a variety of media resources to be placed on the rendered page. The resource may be specified as an image policy or a device depended mime type. Object also allows the control of media transcoding to ensure the resource will render on the requesting agent. See References to media for details.

template:binding

Specifies a binding to a parameter value. If a value is fixed, then it should be set using the value attribute. If value is dynamic or larger than can be used as an attribute, then it can be set using a nested element appropriate to the parameter type.

template:body

The container for the template body.

template:complexValue

Specifies a complex value for a parameter. The element has an open content model.

template:declarations

Contains one or more parameter declarations.

template:definition

Container for template elements.

template:documentation

Container for user documentation. A vocabulary such as XHTML is recommended. A custom XSL stylesheet may be used to extract the content for a user guide.

template:parameter

Defines a parameter. If the default value of the parameter is fixed, it should be set using the default attribute. If it is dynamic, or larger than can be used as an attribute, then it can be set using a nested element appropriate to the parameter type.

template:value

Identifies the position to insert parameter content in the document. This may be used at any point within the body of the template. If the parameter has not been declared, or is not of simple type then an error is raised.

ul

A list of unordered items.

Core attributes

Attributes that are common to XDIME 2 elements.

Hypertext attributes

Attributes that define link targets.
