Common elements

An XDIME 2 page has a well defined structure with several common elements. It must start with a html element containing namespace declarations for the vocabularies in use. The MCS layout and theme policies bound to an XDIME 2 page should be declared in the head section using the link element. The required title element defines the title of the page.

XDIME 2 page structure

  1. The XDIME 2 page needs to start with an XML declaration 
    <?xml version="1.0" encoding="UTF-8"?>
  2. The html element should declare of all the namespaces used on the page. The XDIME 2 elements based on XHTML do not require a namespace prefix. However, there are some groups of elements, such as widgets, that do require prefix.
    <html
  xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs"
  xmlns:template="http://www.volantis.com/xmlns/marlin-template"
  xmlns:widget="http://www.volantis.com/xmlns/2006/05/widget"
  xmlns:si="http://www.volantis.com/xmlns/2004/06/xdimecp/interim/si"
  xmlns:xf="http://www.w3.org/2002/xforms">
  3. The required title element defining the title of the page must be placed within the head element
    <head>
  <title>Client Framework sample application</title>
  <link rel="mcs:theme" href="/main.mthm"/>
  <link rel="mcs:layout" href="/main.mlyt"/>
</head>
  4. The theme and layout policies bound to the XDIME page should be declared in the head section of the page using the link element
  5. The body element contains the body of the XDIME page

XDIME 2 in layoutsÂ 

A single XDIME page structure can be defined by several layout polices.

  1. For example, the main.mlyt layout defines the main structure of the page via the link element
    <link rel="mcs:layout" href="/main.mlyt"/>
  2. One of the elements specified by main.mlyt is the region named 'content'. The 2x2.mlyt layout, declared in the body of the page and bound to the 'content' region, defines its additional structure in the style attribute. The attribute value 'mcs-layout' is equivalent to the Target layout property in the Layout category of the Style Rules editor.
    <div style="mcs-layout: 'layouts/2x2.mlyt';">
  ...
</div>

The table provides links to the element reference sections of the XDIME 2 documentation.

NamePurpose
body

Contains the document's content.

head

Information such as the title and other metadata that is not document content. The contained title element is required.

html

The container for the XDIME 2 document.

link

Defines a single link. Multiple links and relationships may be used in the head section of a document.

title

The title element is used to identify the document. It can contain only text.
