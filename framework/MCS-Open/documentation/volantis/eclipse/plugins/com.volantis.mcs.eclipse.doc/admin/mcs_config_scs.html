Configuring SCS

The configuration of the Secure Connection Server (SCS) host for MCS is set in the mcs-config.xml file.

The optional http-proxy element defines the proxy settings for a secure connection server, and sets the system wide Java properties controlling proxies. If it is not specified, then MCS will not attempt to configure the system properties.

<http-proxy
  host="proxy"
  port="8000"/>

The ports on which the proxy listens for incoming proxy and connection requests, the authentication provider, and logging configuration are defined in the scs-config.xml file.

The proxy-configuration element defines the characteristics of the HTTP/HTTPS proxy. The port attribute defines the port on which SCS listens for requests, and will match the port from the http-proxy element in the mcs-config.xml file. The gateway-timeout attribute defines the length of time in seconds that a request will remain outstanding while a response is awaited from the publishing servers . The secure-connection-configuration element defines the characteristics of the SCS/SPS connection. The port attribute defines the port on which the SCS listens for SPS requests. The keep-alive attribute defines the maximum length of time in seconds between SCS sending a keep alive request to its publishing servers. The authentication-provider element class attribute defines the class that implements the AuthenticationProvider interface. The log4j element points to a log4j.xml file defining the logging appenders.

<scs-config>
   <proxy-configuration
      port="8000"
      gateway-timeout="30"/>
    <secure-connection-configuration
      port="1067"
      keep-alive="120"/>
    <authentication-provider
      class="com.volantis.vdp.authentication.SampleAuthenticationProvider"/>
    <log4j
      xml-configuration-file="/[path_to_scs]/config/scs-log4j.xml"/>
</scs-config>
