MCS views

MCS provides the following views:

  • Device Repository
  • Format Attributes
  • Style Rules

In addition MCS makes use of five workbench views:

  • Navigator
  • Outline
  • Package Explorer
  • Problems
  • Search

You use the Navigator or the Package Explorer views to manage projects, folders and project resources.

The Outline and Format Attributes views work in conjunction with the MCS Layout editor. You can also use the Outline view with a device tree when the Device Repository editor is open. MCS uses the Problems view to report on the errors status of all the resources in your MCS project.

The Style Rules view provides an alternative way to view and refactor all the rules for a particular theme variant.

You can use the Search view to quickly find devices in the Device Repository. When the search results are listed you can open the Device Repository editor on a selected device.

Errors shown in the Problems view for all projects are linked to listings in the policy editors and show in icons in the Navigator and Package Explorer views.
