Defining a template

In this module we will create a template file for the Jive web application. It will define common elements to be used to render pages.

Defining templatesÂ 

  1. An XDIME template starts with the template:definition element. It must contain the namespace declaration for the template prefix.
    <template:definition xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs"
  xmlns:template="http://www.volantis.com/xmlns/marlin-template">
  2. The template:documentation element should be used for user documentation
  3. The template:declarations element contains template parameters. The following example specifies two parameters. First one defines the name of a page. The 'simple' type of that parameter allows character data only. Second parameter defines the page content. The 'complex' type means it may contain character data and an XDIME markup. 
    <template:declarations>
  <template:parameter name="title" type="simple"/>
  <template:parameter name="content" type="complex"/>
</template:declarations>
  4. The template:body element defines the structure of the page 
    <template:body>
  <div id="logo">
    <a class="homelogo" href="#">
      <object class="homelogo" src="/images/jive_logo.mimg"/>
    </a>
  </div>
  <div id="photos">
    <object src="/images/photos.mimg"/>
  </div>
  <!-- The main menu -->
  <div id="menu_company_news" class="main_menu">
    <a class="pnav" href="#">
      <object src="/images/menu_company_news.mimg"/>
    </a>
  </div>
  <div id="menu_human_resources" class="main_menu">
    <a class="pnav" href="#">
      <object src="/images/menu_human_resources.mimg"/>
    </a>
  </div>
  <div id="menu_employee_store" class="main_menu">
    <a class="pnav" href="#">
      <object src="/images/menu_employee_store.mimg"/>
    </a>
  </div>
  <!-- The current stock price -->
  <div id="stockprice"> Latest from NYSE for Jive (VBE): 21.50 (+ 2.25) </div>
  <h2 id="title">
    <template:value ref="title"/>
  </h2>
  <div id="content">
    <template:value ref="content"/>
  </div>
  <div id="footer">
    <p>Copyright <a href="#">Jive Sports</a></p>
  </div>
</template:body>
  5. The <div id="logo"> and <div id="photos> elements specify the Jive logo and sports-related imagery. The src attributes point to the corresponding image component policies.
  6. The <div class="main_menu"> elements define the menu bar
  7. The <div id="stockprice> element displays the stock price
  8. The <h2 id="title"> element specifies the title of the page. The value of the <template:value ref="title"> parameter will be specified in an XDIME file using the template:binding element. Refer to XDIME page with template for further information.
  9. The <div id="content"> element defines the body of the page. Markup from an XDIME page contained by <template:binding name="content"> will be placed here.
    <template:binding name="content">
  <template:complexValue>
    ...
  </template:complexValue>
</template:binding>
  10. The <div id="footer"> element contains the footer

Complete XDIME

  1. Create a template.xdtpl file and save it in the Volantis/MCS/webapps/mcs/tutorial/templates/ directory
  2. Modify the template.xdtpl file by including the following code 
    <?xml version="1.0" encoding="UTF-8"?>

Checklist

NamePurpose
a

Used to create an explicit link to another place in the same document or to another document. The current document is the source of the link and the value of the href attribute, defines the link target

div

A section used to add extra structure to documents. Style sheets can be used to control the presentation.

h1, h2, h3, h4, h5, h6

Heading elements for sections within a document. The number associated with each element indicates its relative position in the hierarchy of headings, with 1 indicating the top level and 6 the lowest level.

object

A reference to a media resource. Object allows a variety of media resources to be placed on the rendered page. The resource may be specified as an image policy or a device depended mime type. Object also allows the control of media transcoding to ensure the resource will render on the requesting agent. See References to media for details.

p

Block element that defines a paragraph.

template:body

The container for the template body.

template:binding

Specifies a binding to a parameter value. If a value is fixed, then it should be set using the value attribute. If value is dynamic or larger than can be used as an attribute, then it can be set using a nested element appropriate to the parameter type.

template:complexValue

Specifies a complex value for a parameter. The element has an open content model.

template:declarations

Contains one or more parameter declarations.

template:definition

Container for template elements.

template:documentation

Container for user documentation. A vocabulary such as XHTML is recommended. A custom XSL stylesheet may be used to extract the content for a user guide.

template:parameter

Defines a parameter. If the default value of the parameter is fixed, it should be set using the default attribute. If it is dynamic, or larger than can be used as an attribute, then it can be set using a nested element appropriate to the parameter type.

template:value

Identifies the position to insert parameter content in the document. This may be used at any point within the body of the template. If the parameter has not been declared, or is not of simple type then an error is raised.
