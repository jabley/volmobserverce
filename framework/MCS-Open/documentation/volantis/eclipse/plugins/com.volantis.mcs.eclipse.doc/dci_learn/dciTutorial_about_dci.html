Adding dynamic content

This tutorial will introduce you to the Dynamic Content Integration (DCI) techniques you can use in developing XDIME 2 pages, incorporating content from multiple sources; for instance, XML documents, web sites or web services.

It provides drivers for different data sources, along with XML vocabularies that specify a series of sequential operations, and the inputs to and outputs from each operation. Such a set of instructions is called a 'pipeline'. For example, there is an XML namespace for accessing web services. In addition, XSL transforms can be used to convert structured information to XDIME. All of this allows pages to be generated dynamically from external content without having to use either Java code or Java Server Pages.

You can also make use of DCI template markup to render pages. Templates can be reused in different contexts with specific parameter values, and are generally easier to specify than equivalent XSL transformations of data sources.
