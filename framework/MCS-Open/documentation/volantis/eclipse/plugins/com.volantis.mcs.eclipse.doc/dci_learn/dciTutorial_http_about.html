Accessing HTTP resources

You can use web driver elements to retrieve content from a web server. The web driver supports the HTTP methods GET and POST to access a remote page, represented by the webd:get and webd:post elements.

In the following modules we will create an XDIME page presenting weather conditions for the Jive Headquarters. The information will be provided by the Yahoo! Weather service, and converted to XDIME using an XSL transform.

HTTP cachingÂ 

The web driver implements most of the HTTP 1.1 caching model, with the principal exception of server response headers containing Etag and Vary headers, which are not supported.

Tip: DCI supports both content and XSLT caching. Refer to Configuring pipeline caches for further information.
