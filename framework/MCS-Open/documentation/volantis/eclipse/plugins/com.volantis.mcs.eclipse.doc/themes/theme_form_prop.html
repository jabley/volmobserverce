Form properties

Purpose

Specifies the appearance and positioning of input and other objects used in forms.

ControlDescriptionOptions
Action StyleHow form action controls should be displayed. The default is usually a button. For WML devices a link, which uses the anchor tag, can be specified instead of a button.default, image, link 
Action ImageImage component or URI to be used when a form action has a Action Style of image. Browse opens the Image Component Selection dialog.none, [image_component], URI 
Indicating ImageImage component or URI to be used next to, or on items such as an anchornone, [image_component] URI 
ColumnsWidth in characters of a text input control 
RowsNumber of rows in a text input control 
Selection List LayoutOrder of the captions and controls in the display if both occupy the same panecaption-first, control-first 
Selection List StyleType of the selection list input controlscontrols, default, circular-list, menu 
Selection List TriggerSpecifies the event that triggers display of a selection listnormal, focus, select 
FocusWhether or not to accept focusaccept, ignore 
HTTP Method HintThe HTTP method to useget, post 
Break afterApplies to xf:group and widget:wizard elements. The default value of 'never' means that the group does not end a form fragment. If set to 'always' then the group ends a form fragment, and another fragment begins immediately.always, never 
Input formatDefines the pattern to be used for validation
