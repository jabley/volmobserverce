XForms model

One of the key elements of the XForms concept is the form model. The XForms model describes the structure of the instance data, specifies the default values and defines where and how to submit the data. A simple XDIME 2 specific data model allows you to initialize form controls and provide additional hidden data during form submission.

The namespace prefix used for elements that originate in XForms is xf: and the namespace declaration is http://www.w3.org/2002/xforms. Two elements specific to forms are defined using the si: prefix, and are used as a simple way of providing instance data to initialize a form. The namespace declaration is http://www.volantis.com/xmlns/2004/06/xdimecp/interim/si.

Form instance dataÂ 

  1. The model definition must be placed within the head element of an XDIME 2 page. The xf:model element contains the model for the form. The following model specifies two form elements: 'formControl1' and 'formControl2'. Please note, that the model does not define their types. It only says that two pieces of information will be collected and submitted using the URL specified by the action attribute. 
    <xf:model id="demoModel">
  <xf:instance>
    <si:instance>
      <si:item name="formControl1">abc123</si:item>
      <si:item name="formControl2"/>
    </si:instance>
  </xf:instance>
  <xf:submission action="test.jsp" id="s1"/>
</xf:model>
  2. The si:instance element is the body element of xf:instance in the model for the form. It can contain one or more si:item elements.
  3. When the name attribute on the si:item element matches the ref attribute on a form control, the content of the si:item element is used to initialize the control. Otherwise, it is treated as additional "hidden" data to be submitted with the form. The name attribute must be unique within a si:instance element. In our case, the form control named 'formControl1' will have its value set to 'abc123' by default.
  4. The xf:submission element specifies the URL to be used when submitting the form. It may optionally contain the method attribute defining the HTTP method by which the form should be submitted. Possible values are: 'get' or 'post'.
  5. The form controls must be specified within the body of the page. The following example illustrates the use of the xf:input element. This element defines a simple text input field. Refer to Text input controls for further information.
    <xf:input model="demoModel" ref="formControl1">
  <xf:label>Label:</xf:label>
</xf:input>
  6. The model attribute identifies the form model that contains the parameter specified in the ref attribute
  7. The xf:label element defines the label for a form control
  8. The xf:submit element initiates submission of the form. Its submission attribute points to the submission element that defines where and how the form should be submitted. See Submitting forms for details.
    <xf:submit submission="s1">
  <xf:label>OK</xf:label>
</xf:submit>

Checklist

NamePurpose
si:instance

Container for data items used to supply initial values for forms controls, or to provide additional data to be submitted with the form.

si:item

Defines an item of instance data that is required if a form control needs to be initialized, or when data not associated with a form control needs to be provided during form submission.

xf:input

A form control for text input.

xf:instance

Optional element that contains instance data associated with the model element. It is used to supply initial values for forms controls, or to provide additional data to be submitted with the form.

xf:label

Provides a descriptive label for forms controls.

xf:model

Represents a form definition, used as a container for elements associated with its submission.

xf:submission

Defines the information to submit, where it should be sent, and how to send it.

xf:submit

A form control that initiates submission of the form data for processing.

Form attributes

Binding expressions provide the mapping between the data associated with a particular control and the data provided for processing when the form is submitted. The expression is the name of the parameter associated with the control's data when the form is submitted. These parameters appear on the submission URL if the form is submitted using the HTTP GET method, and are hidden if the POST method is used. The methods are specified as attributes on the xf:submission element.
