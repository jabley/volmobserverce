Aligning form controls

In this module we will explain how to use layouts to control the position of form elements and their labels on a page. We will start by creating a new layout policy. Then you will learn how to target XDIME 2 content to different layout elements. Refer to Next steps with MCS for further information about using layouts.

Tip: The MCS policy editors operate within the open-source Eclipse Framework. See for details.

Adding a layout 

  1. Choose File | New and pick the Layout wizard from the Eclipse menu
  2. The layout policy should be located in the xforms/WebContent/mcs-policies directory. Choose that folder in the policy tree view, or enter its full path in the first text box.
  3. Name a new layout 'form' and then click the Next button. The MCS wizard automatically adds the .mlyt layout file extension to the file name.
  4. Choose Default in the Selection Criteria box
  5. Click the Finish button to complete

Layout editor 

The Layout Editor has two pages, an Overview page listing the device layouts and their general attributes, and a Design page where you can develop your layout.

Working with layouts 

  1. Make sure the form.mlyt layout is opened in the Layout editor
  2. Pick the 'Default' variant in the list of variants and click on the 'Design' tab at the bottom of the window
  3. Right click in the empty cell area and choose Add | Grid | N by M to open the Add Grid dialog
  4. Enter values for 2 rows and 2 columns and click OK. There are now empty cells for each row.
  5. Add required panes. Right click in the empty cell area and choose Add | Pane | Pane. Repeat this for all four cells.
  6. Name the newly created panes. Your final layout structure should look like the one presented below.

Targeting XDIME 2 content

  1. You use the mcs-container property to target XDIME 2 content to specific layout elements
    <xf:input model="demoAlign" ref="input" style="mcs-container: 'top-right'">
  <xf:label style="mcs-container: 'top-left'">Enter Your Name:</xf:label>
</xf:input>
<xf:select1 model="demoAlign" ref="select1"
  style="mcs-selection-list-style: controls; 
  mcs-selection-list-option-layout: control-first; mcs-container: 'bottom-right'">
  <xf:label style="mcs-container: 'bottom-left'">Select1:</xf:label>
  <xf:item>
    <xf:label>Item1</xf:label>
    <xf:value>Item1</xf:value>
  </xf:item>
  <xf:item>
    <xf:label>Item2</xf:label>
    <xf:value>Item2</xf:value>
  </xf:item>
  <xf:item>
    <xf:label>Item3</xf:label>
    <xf:value>Item3</xf:value>
  </xf:item>
</xf:select1>
  2. The label of the text input field will be displayed in the 'top-left' pane, while the control in the 'top-right' pane. The selection elements and their label will be displayed in the 'bottom-right' and 'bottom-left' pane respectively.

Complete code

  1. Create a new file called xforms_align.xdime
  2. Modify xforms_align.xdime by including the following code 
    <?xml version="1.0" encoding="UTF-8"?>
<html xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs"
  xmlns:si="http://www.volantis.com/xmlns/2006/01/xdime2/si"
  xmlns:xf="http://www.w3.org/2002/xforms">
  <head>
    <title>Aligning Form Controls</title>
    <link rel="mcs:layout" href="/form.mlyt"/>
    <xf:model id="demoAlign">
      <xf:instance>
        <si:instance>
          <si:item name="input"/>
          <si:item name="select1"/>
        </si:instance>
      </xf:instance>
    </xf:model>
  </head>
  <body>
    <xf:input model="demoAlign" ref="input" style="mcs-container: 'top-right'">
      <xf:label style="mcs-container: 'top-left'">Enter Your Name:</xf:label>
    </xf:input>
    <xf:select1 model="demoAlign" ref="select1"
      style="mcs-selection-list-style: controls; 
      mcs-selection-list-option-layout: control-first; mcs-container: 'bottom-right'">
      <xf:label style="mcs-container: 'bottom-left'">Select1:</xf:label>
      <xf:item>
        <xf:label>Item1</xf:label>
        <xf:value>Item1</xf:value>
      </xf:item>
      <xf:item>
        <xf:label>Item2</xf:label>
        <xf:value>Item2</xf:value>
      </xf:item>
      <xf:item>
        <xf:label>Item3</xf:label>
        <xf:value>Item3</xf:value>
      </xf:item>
    </xf:select1>
  </body>
</html>

Checklist

NamePurpose
si:instance

Container for data items used to supply initial values for forms controls, or to provide additional data to be submitted with the form.

si:item

Defines an item of instance data that is required if a form control needs to be initialized, or when data not associated with a form control needs to be provided during form submission.

xf:input

A form control for text input.

xf:instance

Optional element that contains instance data associated with the model element. It is used to supply initial values for forms controls, or to provide additional data to be submitted with the form.

xf:label

Provides a descriptive label for forms controls.

xf:model

Represents a form definition, used as a container for elements associated with its submission.

xf:select1

A form control that provides for selection of a single choice from an available list of options; for example, a group of radio buttons.

Form attributes

Binding expressions provide the mapping between the data associated with a particular control and the data provided for processing when the form is submitted. The expression is the name of the parameter associated with the control's data when the form is submitted. These parameters appear on the submission URL if the form is submitted using the HTTP GET method, and are hidden if the POST method is used. The methods are specified as attributes on the xf:submission element.
