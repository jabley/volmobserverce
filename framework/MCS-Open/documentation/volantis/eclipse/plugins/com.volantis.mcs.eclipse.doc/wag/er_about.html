About element references

All the element references relevant to working with MCS have a uniform structure.

Elements

Each element is introduced with a short description. Links to contained and containing block elements follow, with the exception of inline elements, and those with complex nested content models.

Attributes

A table lists element attributes. Each has a short description, the XML schema datatype, and an indication whether the attribute is optional, required or deprecated. Any default values, and predefined options are added where relevant.

Required attribute names are highlighted in bold type.

Attributes that are common to several elements are grouped together on separate pages, with links from the related element descriptions, rather than being repeated throughout.

Examples

In most cases, an example shows how the element is used.
