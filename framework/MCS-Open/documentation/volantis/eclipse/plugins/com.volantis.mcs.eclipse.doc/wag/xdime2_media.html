References to media

In XDIME 2 you can use an object element to support audio, flash and video media types, in addition to the image type.

You can use device independent policies, file types .mauc, and .mdyv, where the selection is based on device or category targeting. However, transcoding is not supported in these cases. You can also process external resources, where transcoding is based on device policies.

Parameters used with media

mcs-aspect-ratio-height
Defines the height part of the aspect ratio of the image
mcs-aspect-ratio-width
Specifies the width part of the aspect ratio of the image
mcs-external-label
Provides the content for a link that will activate the display of an external resource
mcs-transcode
Whether or not an image is transcoded. It takes the values 'true' (default) or 'false'. If the reference in the src attribute is to an image file then the image will be transcoded. If it is to an MCS image component, the attribute is ignored, since in MCS it will already be typed as a convertible image.
Note: Flash objects are not transcoded, since they will scale automatically on the client. The mcs-transcode parameter must therefore be set to false. Otherwise an error will occur.

Using the parameters

The parameter values are provided in the name and value attributes of the contained param element.

One way to use the parameters is to supply the actual size of the source image that will be used for transcoding. If the source image is 320 pixels wide and 180 pixels high, the following markup could be used.

<object src="someURI">
  <param name="mcs-aspect-ratio-width" value="320"/>
  <param name="mcs-aspect-ratio-height" value="180"/>
</object>

However, it is not actually necessary to know the exact size of the source image. For example, the alternative markup yields the same result in most cases.

<object src="someURI">
  <param name="mcs-aspect-ratio-width" value="16"/>
  <param name="mcs-aspect-ratio-height" value="9"/>
</object>

It is the relative values of width and height that are important. These allow ICS to calculate the final size of the image in a way most appropriate for the device.

Tip: If possible you should provide values for the true image size, because this will improve accuracy in cases where an image might not be scaled to a whole number of pixels.

Authors can specify the mcs-external-label parameter, which defines the content used for a link to an external resource, without having to know the exact media type type.

<object
  src="http://example.com/sports/goal.png"
  mcs-media-style="external">
  <param name="mcs-external-label" value="Last night's winning goal"/>
</object>

Styling

You use the mcs-media-style property to determine whether an object element should be treated as part of a page or an external resource that a user will explicitly retrieve.

The property has two possible values; 'integral' means that the media should be displayed automatically, and 'external' means that a user action is required.

Any value that is not supported by the device is ignored, and the first remaining value, if any, determines the markup to generate. If there are no remaining values then the object element is ignored and its body is processed instead.

Specifying the source type

If you use the srctype attribute on the object element, then you must refer to an MCS policy variant, and all the variants must be of the same type.

If the srctype attribute is not specified, MCS will attempt to work out the type based on the file extension using values in the web.xml file.
