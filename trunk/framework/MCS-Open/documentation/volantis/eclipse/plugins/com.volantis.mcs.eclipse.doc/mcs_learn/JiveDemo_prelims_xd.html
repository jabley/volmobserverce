XDIME preliminaries

An XDIME 2 page has a well defined structure. It must start with a html element containing namespace declarations for the vocabularies in use. The majority of XDIME 2 presentation elements do not require a namespace prefix. However, there are some groups of elements, such as XForms, that do require prefix.

The MCS layout and theme policies bound to an XDIME 2 page should be declared in the head section using the link element. The required title element defines the title of the page.

The head element may contain additional elements, such as xf:model for example.

Declaring XML namespaces

  1. Open your XML editor, create a new file, and save it as jivehome.xdime in the jive project folder
  2. Enter the following markup shown below which is a typical namespace declaration. 
    <?xml version="1.0" encoding="UTF-8"?>
<html xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs"
  xmlns:xf="http://www.w3.org/2002/xforms"
  xmlns:sel="http://www.w3.org/2004/06/diselect">
  ...
</html>

Binding to the layout and theme

  1. Enter the code shown below to bind the jivehome.mlyt layout and the jive.mthm theme to this page called 'Welcome to the Jive Intranet' 
    <head>
  <title>Welcome to the Jive Intranet</title>
  <link rel="mcs:layout" href="/jivehome.mlyt"/>
  <link rel="mcs:theme" href="/jive.mthm"/>
</head>
  2. Save you work, and continue to the next module to add common element features.

Checklist

Here are some reference notes of the XDIME elements you have met so far. Click the name to see the full reference.

NamePurpose
head

Information such as the title and other metadata that is not document content. The contained title element is required.

html

The container for the XDIME 2 document.

link

Defines a single link. Multiple links and relationships may be used in the head section of a document.

title

The title element is used to identify the document. It can contain only text.
