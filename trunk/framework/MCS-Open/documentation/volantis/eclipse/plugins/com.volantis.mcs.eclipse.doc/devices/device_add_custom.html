Adding a custom device policy

On the Structure page of the Device Repository editor you can provide the definitions for custom policy type, available values, and associated meta-information that combine to make a device repository policy.

Once you have created the new policies, they will be available in other policy editors for targeting using the Device category policy dropdown in the Attributes section.

  1. In the Device tree Policies section, select Custom (or a policy under Custom) as a fallback. MCS enables the New button.
  2. Click New to start the New Policy wizard
  3. Enter a name in the Name text box. Policy names must be unique in a repository, so if you use the name of an existing policy you'll see a warning.
  4. Choose a combination of Composition and Type for the policy. The options in Type will change depending on the value in Composition.
  5. Click Finish to close the wizard
  6. In the Policy Definition section, Add the Select values that you want for a Selection type, and use the Up or Down buttons to order your list. If you chose a 'Range' type enter the values for From and To.
  7. Add a user-friendlyName. MCS displays this text in the device tree and the Master Value section.
  8. Enter a Help text that will appear as a tool-tip for the policy controls
  9. Define a value in the Master Value section
  10. Add a UAProf name

MCS displays the policy control you have defined the next time you go to the Policies page and select the custom category.

To rename a custom device, right click on the policy, choose Rename and enter the new name in the dialog.
