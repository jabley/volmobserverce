Overriding device character sets

On some devices, the default output character set may not support certain character entities like &bull;, and &nbsp;.

To work around this problem, you can make an entry in the device repository that will override the default character set defined by the device.

  1. Open the Device Repository editor on the current repository
  2. Go to the Policies page and select the Output category
  3. Next to the Output charset (forced) choose Override
  4. Enter the override value
Note: If you experience device issues of this kind, you should report them to your support desk for referral to the Device Update Service.
