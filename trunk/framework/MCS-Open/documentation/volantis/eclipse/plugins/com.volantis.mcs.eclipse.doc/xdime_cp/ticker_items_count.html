ticker:items-count

Purpose

Returns the number of items held by the BuzzCast client.

Contained by

body

Attributes

AttributeDescriptionTypeDefaultOptionsUse
channelSelects items from the single channel only. If attribute is not specified, all items are selected.  optional 
followedSelects items by the 'followed' status before counting. If set to 'yes', only items for which a link have been followed are selected. If the attribute value is set to 'no', only items for which no link has been followed are selected. If attribute is not specified, all items are selected. yes, no optional 
readSelects items by the 'read' status before counting. If the attribute is set to 'yes', only items that have been read are selected. If the attribute value is set 'no', only unread items are selected. If attribute is not specified, all read and unread items are selected. yes, no optional 

Attribute group

Core attributes
