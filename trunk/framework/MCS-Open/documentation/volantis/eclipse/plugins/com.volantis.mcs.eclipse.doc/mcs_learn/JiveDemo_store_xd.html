Employee store page

This page illustrates the use of a XForms elements. The namespace prefix used for elements that originate in XForms is xf: and the namespace declaration is http://www.w3.org/2002/xforms.

In our example, these elements create a form that allows employees to check the availability of items for purchase.

XForms model

  1. Create a employeeStore.xdime file in the main jive folder
  2. Add the forms namespace declaration to the html 
    <?xml version="1.0" encoding="UTF-8"?>
<html xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:mcs="http://www.volantis.com/xmlns/2006/01/xdime/mcs"
  xmlns:xf="http://www.w3.org/2002/xforms"
    ...
</html>
  3. Within the head element the required xf:model element defines the model for the form. In the action, it contains the definition of the URL to be used when the form is submitted. The form data is processed by the JSP checkAvail.jsp, described in the next module. Add the following markup. 
    <head>
  <link rel="mcs:layout" href="/jivearticle.mlyt"/>
  <link rel="mcs:theme" href="/jive.mthm"/>
  <xf:model>
    <xf:submission id="avail" action="checkAvail.jsp"/>
  </xf:model>
</head>

XForms controls

  1. The form controls appear within the body of the page. The xf:select1 elements define two single selection dropdowns. One of these contains a list of products, the other a list of sizes. The ref attribute contains the value of the selection. xf:item elements define labels and values for the clothing types. Complete the item section now. You can skip the content for sizes until later.
    <h3 class="company_header">Availability</h3>
<p>
  <xf:select1 ref="item">
    <xf:label>Item</xf:label>
    <xf:item>
      <xf:label>Storm Jacket</xf:label>
      <xf:value>storm jacket</xf:value>
    </xf:item>
    <xf:item>
      <xf:label>Long Sleeve Sport</xf:label>
      <xf:value>long sleeve sports shirt</xf:value>
    </xf:item>
    <xf:item>
      <xf:label>Twill Shirt</xf:label>
      <xf:value>twill shirt</xf:value>
    </xf:item>
    <xf:item>
      <xf:label>Jersey Sport Shirt</xf:label>
      <xf:value>jersey sports shirt</xf:value>
    </xf:item>
    <xf:item>
      <xf:label>Cotton Sport Shirt</xf:label>
      <xf:value>cotton sports shirt</xf:value>
    </xf:item>
  </xf:select1>
</p>
  2. The xf:submit element defines the Check button. In the submission attribute, it references the xf:submission element identifier in the model you defined. 
    <p>
  <xf:submit submission="avail">
    <xf:label>Check</xf:label>
  </xf:submit>
</p>

Checklist

Take some time to review the structure of the XForms elements in detail by following the links below.

NamePurpose
div

A section used to add extra structure to documents. Style sheets can be used to control the presentation.

h1, h2, h3, h4, h5, h6

Heading elements for sections within a document. The number associated with each element indicates its relative position in the hierarchy of headings, with 1 indicating the top level and 6 the lowest level.

p

Block element that defines a paragraph.

xf:item

Container for the option values in xf:select and xf:select1 elements.

xf:label

Provides a descriptive label for forms controls.

xf:model

Represents a form definition, used as a container for elements associated with its submission.

xf:select1

A form control that provides for selection of a single choice from an available list of options; for example, a group of radio buttons.

xf:submission

Defines the information to submit, where it should be sent, and how to send it.

xf:submit

A form control that initiates submission of the form data for processing.

xf:value

Provides the option values associated with select controls.

Core attributes

Attributes that are common to XDIME 2 elements.

Form attributes

Binding expressions provide the mapping between the data associated with a particular control and the data provided for processing when the form is submitted. The expression is the name of the parameter associated with the control's data when the form is submitted. These parameters appear on the submission URL if the form is submitted using the HTTP GET method, and are hidden if the POST method is used. The methods are specified as attributes on the xf:submission element.
