pipeline:convertImageURLToTranscoder

Purpose

Rewrites all img element URLs in a page to be routed through the Media Access Proxy (MAP). The operation will calculate the most appropriate transcoding rule to use, based on the calling device characteristics.

Attributes

AttributeDescriptionTypeDefaultOptionsUse
maxfilesizeThe maximum image sizexs:integernone  optional 
relativeWidthThe percentage width of the image on the devicexs:integernone  optional 
serverThe address of the proxy serverxs:anyURInone  required 

Examples

<html xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:urid="http://www.volantis.com/xmlns/marlin-uri-driver"
  xmlns:pipeline="http://www.volantis.com/xmlns/marlin-pipeline">
  <head>
    <title>pipeline:convertImageURLToTranscoder</title>
  </head>
  <body>
    <div>
      <pipeline:convertImageURLToTranscoder server="http://ics.mysite.com:8080" maxfilesize="9000"
        relativeWidth="65">
        <urid:fetch href="test.xdinc"/>
      </pipeline:convertImageURLToTranscoder>
    </div>
  </body>
</html>

The test.xdinc file should contain the following code.

<?xml version="1.0" encoding="UTF-8"?>
<div xmlns="http://www.w3.org/2002/06/xhtml2">
  <p>Volantis Systems</p>
  <object src="http://localhost:8080/mcs/welcome/images/logo.gif" srctype="image/gif">
    <param name="mcs-transcode" value="true"/>
  </object>
</div>
