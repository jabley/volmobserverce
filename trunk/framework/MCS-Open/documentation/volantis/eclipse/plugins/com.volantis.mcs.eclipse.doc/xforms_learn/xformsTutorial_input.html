Text input controls

The most commonly used inputs in forms are the text input controls. MCS supports three types of such elements, namely xf:input, xf:secret and xf:textarea.

xf:input 

  1. The xf:input element defines a simple text input field 
    <xf:input model="demoInput" ref="input">
  <xf:label>Label:</xf:label>
</xf:input>

  2. The model attribute points to the form model defined within the head section of the page
  3. The ref attribute references the element specified by the model
  4. The xf:label element defines the label for the form control

xf:secret 

  1. The xf:secret element provides a special type of the input control intended to input passwords or other sensitive information. The secret control usually masks entered values with the '*' characters. 
    <xf:secret model="demoInput" ref="secret">
  <xf:label>Label:</xf:label>
</xf:secret>

  2. The model and ref attributes bind the control and the form model. See XForms model for details.

xf:textarea 

  1. The user may want to enter more text than allowed by a single line input field. You use xf:textarea to provide a multi-line text input control. 
    <xf:textarea model="demoInput" ref="textarea" style="mcs-rows: 3; mcs-columns: 20">
  <xf:label>Label:</xf:label>
</xf:textarea>

  2. The mcs-rows and mcs-columns properties define the size of the textarea. For the sake of clarity those style properties are specified using the style attribute. However, it is a good practice to use themes to control styling. Refer to the MCS documentation for further information about theme policies.

Complete XDIME 2 code

The following code illustrates the use of the text input controls.

  1. Create a new XDIME file called xforms_input.xdime, and modify it by including the following code 
    <?xml version="1.0" encoding="UTF-8"?>
<html xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:si="http://www.volantis.com/xmlns/2006/01/xdime2/si"
  xmlns:xf="http://www.w3.org/2002/xforms">
  <head>
    <title>Text Input Controls</title>
    <xf:model id="demoInput">
      <xf:instance>
        <si:instance>
          <si:item name="input">abc123</si:item>
          <si:item name="secret"/>
          <si:item name="textarea"/>
        </si:instance>
      </xf:instance>
    </xf:model>
  </head>
  <body>
    <h4>Text Input</h4>
    <p>
      <xf:input model="demoInput" ref="input">
        <xf:label>Label:</xf:label>
      </xf:input>
    </p>
    <h4>Secret Input</h4>
    <p>
      <xf:secret model="demoInput" ref="secret">
        <xf:label>Label:</xf:label>
      </xf:secret>
    </p>
    <h4>Textbox</h4>
    <p>
      <xf:textarea model="demoInput" ref="textarea" style="mcs-rows: 3;
        mcs-columns: 20">
        <xf:label>Label:</xf:label>
      </xf:textarea>
    </p>
  </body>
</html>

Checklist

NamePurpose
si:instance

Container for data items used to supply initial values for forms controls, or to provide additional data to be submitted with the form.

si:item

Defines an item of instance data that is required if a form control needs to be initialized, or when data not associated with a form control needs to be provided during form submission.

xf:input

A form control for text input.

xf:instance

Optional element that contains instance data associated with the model element. It is used to supply initial values for forms controls, or to provide additional data to be submitted with the form.

xf:label

Provides a descriptive label for forms controls.

xf:model

Represents a form definition, used as a container for elements associated with its submission.

xf:secret

Used for input of confidential or sensitive information such as passwords.

xf:textarea

A form control for larger amounts of text. Multi-line input, such as the body of an message, is an example.

Form attributes

Binding expressions provide the mapping between the data associated with a particular control and the data provided for processing when the form is submitted. The expression is the name of the parameter associated with the control's data when the form is submitted. These parameters appear on the submission URL if the form is submitted using the HTTP GET method, and are hidden if the POST method is used. The methods are specified as attributes on the xf:submission element.
