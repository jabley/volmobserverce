Common page elements

The XDIME 2 pages described in the next modules contain some common div elements, which we will discuss here. The figure shows the PC and Smartphone versions of the Jive home page. Note that although they look differently, the XDIME 2 markup is exactly the same. The differences are achieved by the combination of image component policies, layouts, and themes.

Header

The PC and Smartphone versions of the banner are quite different. The PC version consists of the Jive logo and separate sports photos. The Smartphone version combines these into a smaller, simpler image. This is achieved through a combination of references to the images, layout and theme that you created earlier.

  1. Open the file jivehome.xdime that you created in the previous module. Have the files jive_logo.mimgjivehome.mlyt and jive.mthm open for reference
  2. In the Theme editor select the PC variant, and on the Design page pick the #logo selector and the layout category. The Target Container property identifies the 'logo' pane in the layout. Associate the theme using the id attribute on the div element. 
    <div id="logo">
  ...   
</div>
  3. The logo contains a link to the home page. Add an a element with the 'homelogo' class attribute value, and jivehome.xdime in the href attribute. 
    <div id="logo">
  <a class="homelogo" href="jivehome.xdime">
  ...   
  </a>
</div>
  4. The logo reference is to the image component you added earlier. Enter an object element with the project relative path in the src attribute. 
    <div id="logo">
  <a class="homelogo" href="jivehome.xdime">
    <object class="homelogo" src="/images/jive_logo.mimg"/>
  </a>
</div>
  5. The photo section applies only to the PC page, and there is no link. Enter the following markup, and wrap both div elements in another div to complete your work on the header. 
    <div>
  <div id="logo">
    <a class="homelogo" href="jivehome.xdime">
      <object class="homelogo" src="/images/jive_logo.mimg"/>
    </a>
  </div>
  <div id="photos">
    <object src="/images/photos.mimg"/>
  </div>
</div>

Menus

The structure of the menus is similar to the header. Each menu is represented by an image. The PC version the image of the text; the Smartphone variant is an icon.

  1. Each menu is assigned to a particular pane. Refer to the jivehome.mlyt layout as a reminder. Again, the value in Target Container property is the value to use in the id attribute. Enter the following markup, referring to companyNews.xdime and menu_company_news.mimg. 
    <div id="menu_company_news" class="main_menu">
  <a class="pnav" href="companyNews.xdime">
    <object src="/images/menu_company_news.mimg"/>
  </a>
</div>
  2. Repeat the process with the Human Resources and Employee Store menus. Your additional markup should look like this.
    <div id="menu_human_resources" class="main_menu">
  <a class="pnav" href="humanResources.xdime">
    <object src="/images/menu_human_resources.mimg"/>
  </a>
</div>
<div id="menu_employee_store" class="main_menu">
  <a class="pnav" href="employeeStore.xdime">
    <object src="/images/menu_employee_store.mimg"/>
  </a>
</div>

Welcome

  1. This section displays a welcome, and the current company stock price (on the PC version only). Apply styling on the id attribute referencing the stockprice selector in the theme. 
    <div id="stockprice">Latest from NYSE for Jive (VBE): 21.50 (+ 2.25) </div>
  2. The title is declared in an h2 and targeted on the 'title' pane in the layout.
    <h2 id="title">Welcome to Jive Intranet</h2>

Footer

  1. An identical footer is used in every page of the demonstration. It provides a common copyright notice and a link back to the home page. 
    <div id="footer">
  <p>Copyright <a href="jivehome.xdime">Jive Sports</a></p>
</div>
  2. Save your work, and continue to the next module to add body elements to the home page. Since this content can be reused later in the project, you may want to keep a copy at this stage.

Checklist

NamePurpose
a

Used to create an explicit link to another place in the same document or to another document. The current document is the source of the link and the value of the href attribute, defines the link target

div

A section used to add extra structure to documents. Style sheets can be used to control the presentation.

h1, h2, h3, h4, h5, h6

Heading elements for sections within a document. The number associated with each element indicates its relative position in the hierarchy of headings, with 1 indicating the top level and 6 the lowest level.

object

A reference to a media resource. Object allows a variety of media resources to be placed on the rendered page. The resource may be specified as an image policy or a device depended mime type. Object also allows the control of media transcoding to ensure the resource will render on the requesting agent. See References to media for details.

p

Block element that defines a paragraph.

Core attributes

Attributes that are common to XDIME 2 elements.

Hypertext attributes

Attributes that define link targets.
