Device policy categories

In the Policies section of the Device Repository view you can examine a list containing device attributes. By clicking on the category column head you can sort the attributes into the categories in the table. In the list, category names begin with lower case and Dynamic Visual is abbreviated to 'dynvisual'.

The same categories are displayed in the Policy page of the Device Repository editor.

Note: It may be necessary to modify or replace device policy properties in the Transitional category, so you should regard them as 'experimental' for the present. These properties cannot be accessed using MCS expressions or through the repository API.

The table describes the categories and gives some examples of attributes.

Category  Description  Examples 
Audio  The device's ability to play audio clips  GSM, MIDI, MP3 
Browser  Basic attributes of the browser  Protocol, browser name 
Custom  A category for implementation-specific policies; not part of the standard device repository. Policies with 'Single' composition and 'Selection' type may be used to target variants   
Download  Audio and video download attributes  3GPP audio downloads, MIDI audio codec 
Dynamic Visual  The device's ability to show dynamic visuals  QuickTime, Real Video 
Ergonomics  Usability factors  Weight, viewing distance 
Identification  Attributes that identify the device  Manufacturer's name, model number 
Image  The device's ability to display images  WBMP, JPEG 
Input  Capabilities relating to text or drawing input, and other inputs such as bar code scanners and video cameras  Type of keyboard, type of pointing device 
Location  Device-based Location detection attributes  The kind of GPS service in use 
Message  Message-related information  XHTML, MMS, SMS 
MIDP  Attributes used when creating XDIME Player packages  Video window height in pixels 
Miscellaneous  Miscellaneous attributes   
Network  Attributes covering network aspects  Assumed bandwidth 
Output  Physical attributes of the primary visual display or other output mechanisms  Physical size of the display, ability to play audio 
Protocol  Attributes used to control protocol-specific behavior in MCS, at the device level  Style substitution for WML devices 
Rules  The short names of the conversion rules to be used  gg1, gp1 
Security  Capabilities relating to the information security of the device or the protocols that it supports  SSL support, strong user authentication support 
Streaming  Audio and video streaming attibutes  ACC audio streaming, MP3 audio codec 
System  Internal hardware attributes such as processor speed and memory capacity  Processor name, clock speed 
Transitional  Parameters that address known markup and CSS rendering issues
