Validating forms

You can use client- and/or server-side validation to improve the quality of data submitted by users. Client-side validation avoids the need to repeatedly send a form to the server for error checking. However, not all devices support a full range of client-side validation features, and some may be incapable of supporting any validation at all. So it is good practice to use server-side form validation in all events.

Validating form input 

Some devices that support XHTML Mobile Profile protocol also support WCSS (Wireless CSS) validation. You use the mcs-input-format theme property to specify the validation pattern, for example <xf:input style="mcs-input-format: "M:A*a"">. Please note, that the " character inside the style definition must be encoded using &quot;.

Use the following code to check if your device supports WCSS validation.

<?xml version="1.0" encoding="UTF-8"?>
<html xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:sel="http://www.w3.org/2004/06/diselect">
  <head>
    <title>WCSS Validation</title>
  </head>
  <body>
    <sel:select>
      <sel:when expr="device:getPolicyValue('protocol.wcss.validation.styles')='true'">
        <p>This device supports WCSS validation.</p>
      </sel:when>
      <sel:otherwise>
        <p>This device does not support WCSS validation.</p>
      </sel:otherwise>
    </sel:select>
  </body>
</html>

Using forms validator widgets 

You may use widgets to validate input in XForms elements. The Simple Forms Validator widget checks if the value in a single XForms field is valid. The Multiple Forms Validator checks that the values in multiple fields, taken together, meet some specified constraints. Refer to Validating a form and Validating multiple forms in the Using the widget library tutorial for further information.

This validation method is limited to devices which support Client Framework. Use the following code to verify if your device supports it.

<?xml version="1.0" encoding="UTF-8"?>
<html xmlns="http://www.w3.org/2002/06/xhtml2"
  xmlns:sel="http://www.w3.org/2004/06/diselect">
  <head>
    <title>Client Framework Support</title>
  </head>
  <body>
    <sel:select>
      <sel:when expr="device:getPolicyValue('supports.client.framework')='true'">
        <p>This device supports Client Framework.</p>
      </sel:when>
      <sel:otherwise>
        <p>This device does not support Client Framework.</p>
      </sel:otherwise>
    </sel:select>
  </body>
</html>
