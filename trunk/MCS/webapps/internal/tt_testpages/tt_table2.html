Table Without Classes

IE5 will color cell borders the same colors as the table border, but NN will always color cell borders black/silver. You cannot change these colors in NN. To make the table appear the same in IE I have therfore specified that (row) borders must be black/silver. (Saves specifying border colors for each cell!)

NB: Scrolling down beyond the table causes NN4 to lose the bottom border (although behavior is erratic.) Don't think we can do anything about this!

The table below appears the same in NN4, NN6 and IE5.

table bgcolor=white table border=cyan, borderlight=aqua, borderdark=teal
cell bgcolor=red cell border=blue
cell align=center cell align=right

 

I have not found a method of making the whole table border a single solid color on NN4. This can be done on NN6 by using a class with a solid border style.

All versions of NN ignores bordercolorlight and bordercolordark attributes.

The table below still has aqua/teal border when viewed in NN4. In NN6 and IE5 the table below has solid aqua border.

table width=75% table align=center
table border solid aqua table border=10, cell spacing=20

 

The table below appears the same in NN4, NN6 and IE5

table border=0 table cellspacing=0, cellpadding=0
cell bgcolor=red cell bgcolor=blue

 

The only way to color the cell spacing magenta in NN4 is to place the whole table on a magenta background. To do this the table is placed inside another table with a single cell that has a magenta background. Each cell is then given a white background where necessary. See the source for this page if necessary.
