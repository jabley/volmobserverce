Form instance data

If you want to initialize form controls or provide additional hidden data during form submission, you use a simple data model that is specific to XDIME 2, rather than an XML instance. These elements take an si: namespace prefix.

The si:instance element is the body content of an xf:instance element in the model for the form. It can contain one or more si:item elements. When the name attribute on the si:item matches the ref attribute on a form control, the content of the si:item element is used to initialize the control. Otherwise, it is treated as additional data to be submitted with the form. The name attribute must be unique within a si:instance element.

The matching process is straightforward for the input elements xf:input, xf:secret and xf:textarea.

In select controls, the si:item content should additionally match one of the possible item values. If the content does not match a valid value, the default selection is used.

For a xf:select1 element, the content is treated as the initial value of the control. In the case of an xf:select element, the body content is interpreted as a comma separated list of values. If a value matches that of one of the items, that item is marked as selected.
