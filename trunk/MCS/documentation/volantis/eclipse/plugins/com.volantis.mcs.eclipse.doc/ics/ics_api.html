Pluggable transcoder interface

The PluggableAssetTranscoder interface allows you to swap in and out different transcoding technologies on for any instance of MCS. The interface supports a single transcoding technology for a single instance. Using the interface you can create the correct URL to invoke the transcoding for a variant.

URL processing

Policy values in the MCS device repository define parameters that are appended to the generated URL.

For TransForce compatibilty and ICS modes the following processing takes place.

  • The rule will be checked to see if it specifies GIF encoding. These rules are gg1, gg2, gg4, gg8 and cg8.
  • If one of these rules is matched then the supported encodings for the device will be checked to see if an other encoding format is allowed. The preference will be to map to the equivalent PNG rule but if this is not supported then WBMP will be considered. The table shows the mapping of GIF rules to alternatives.

For all modes the following additional processing takes place.

  • The processed [rule] is inserted after the last '/' in the URL
  • A '/' is inserted immediately after the ruleValue
  • A [width] value is added with '?' or '&' as appropriate
  • A [maxfilesize] value is appended to the URL if it is greater than zero
  • Policy value attributes are appended. If they are not null and do not start with an '&' then this is added automatically.

The table shows the mapping of GIF rules to alternatives, and the values of [width] and [maxfilesize] overrides for each mode.

Mode GIF rule Alternative rule Width Size
Transforce native      tf.width  tf.maxfilesize 
Transforce compatibility, ICS  gg1  gp1 or gw1  vt.width  vt.maxfilesize 
gg2  gp2 or gw1 
gg4  gp4 or gw1 
gg8  gp8 or gw1 
cg8  cp8 or gw1
