About ICS

In the Volantis Mobile Content Framework, Multi-Channel Server (MCS) uses the Image Conversion Service (ICS) to handle image transcoding. ICS is also able to run in TransForce compatibility mode, allowing users to migrate from PictureIQ's TransForce product.

ICS supports the use of convertible image variants. XDIME authors can also use the url and urlc attributes on image and logo elements to specify a transcodable URL in XML content. ICS passes through the URL in the case of url, or adds a width parameter derived from the layout policy in the case of urlc.

Each set of convertible image variants resident in a particular location should be assigned an asset group. The actual images do not need to reside on the ICS server. The asset group must indicate the URL of the ICS server, not the variant location itself. The server for the actual image variants may be indicated using appropriate URL parameters or the default image host will be used as specified in the ics-config.xml file. If using a non-default image host, the image host name should be provided using either the v.imgHost or the tf.source.host parameter.

When a convertible image variant is rendered at runtime, the variant URL is created dynamically using the asset group, variant value and the associated transcoding rule specified for the target device in the MCS device repository.

After ICS is installed you should:

  • Review the features of ICS
  • Decide on deployment options, and consider some performance issues
  • Configure the ICS parameter and log settings
  • Optionally install ICS to another server

Note: Because file locations are set during installation, the configuration examples shown in related topics use placeholders for parts of the directory structure. You should substitute [path] with the path preceeding the location described, and [context_root] with the application root context.
