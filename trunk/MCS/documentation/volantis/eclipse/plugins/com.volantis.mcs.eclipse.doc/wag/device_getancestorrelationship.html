device:getAncestorRelationship

Purpose

Checks whether the named device is an ancestor of the current device. It is equivalent to the getAncestorRelationShip method of the MarinerRequestContext class. The return codes from that method are mapped to one of the returned values.

Signature

device:getAncestorRelationship($device as xs:string) as xs:string?

Parameters

NameDescriptionType
ancestor

The device is an ancestor of the current device

xs:string
device

As a called value, the name of the device to compare against

xs:string
device

As a returned value, the named device is the same as the current device

xs:string
unknown

The device is unknown, or there is no current device

xs:string
unrelated

The device is unrelated to the current device in that it is not actually an ancestor

xs:string

Example

<p expr="device:getAncestorRelationship('PC') = 'ancestor'
or device:getAncestorRelationship('PC') = 'device'">
This message is only for those PC users out there.
</p>
