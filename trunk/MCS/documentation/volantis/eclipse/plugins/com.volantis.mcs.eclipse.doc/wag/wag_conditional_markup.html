Using conditional markup

Conditional expressions provide the means for testing values against one another and using markup accordingly. You can use a group of 'select' elements on larger blocks of content, and the expr attribute on individual elements.

Select elements

The select, when and otherwise elements provide a way to include content that is similar in use to case and switch statements in procedural languages.

<select precept="matchfirst">
   <when expr="device:getPolicyValue('pixelsx')='300'">
      <img src="/bigImage.mimg"/>
   </when>
   <when expr="device:getPolicyValue('name')='Nokia-7650'">
      <img src="/littleImage.mimg"/?
   </when>
   <otherwise>
      <p>The Image</p>
   </otherwise>
</select>

Using the expr attribute

All XDIME elements with the exception of otherwise have an expr attribute. It is optional on all elements except the select and when elements, where it is used differently.

expr can contain an MCS expression. If it evaluates to true the element is processed as normal. If the expression evaluates to false then the element and its children are skipped, and processing continues.

The example shows that MCS will only process the pane element and its children if a function device:getPolicyValue('java') returns the value 'J2ME'.

<canvas>
   <pane name="main" expr="device:getPolicyValue('java')='J2ME'">
      <p>Your browser supports J2ME</p>
   </pane> 
</canvas>
