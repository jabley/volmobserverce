Developing MCS policies

This overview will help you to understand the best approaches to using MCS to process device independent markup. For details on how to use the Eclipse interface to create policies refer to Working with MCS policies.

Working with device independent principles offers benefits, but it requires careful planning if the benefits are to be realized. There are a number of issues to consider ranging from project structure, through to team working arrangements.

Planning your application

Before you start to develop policies you should

  • Read about the concept of device independence and how MCS handles a range of delivery contexts by separating content from presentation
  • Understand how to make the best use of MCS in the Eclipse environment
  • Learn how MCS stores policies in repositories
  • Consider the individual and team development options
