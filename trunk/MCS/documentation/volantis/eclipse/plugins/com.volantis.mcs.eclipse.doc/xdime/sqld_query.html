sqld:query

Purpose

Container for a SQL query. The query can be a normal query, or be configured to take parameters. If a parameter is used, the query element must contain a nested string parameter.

Note: The transaction attribute binds the SQL statement to the transaction. Although it is optional the transaction attribute should always be used when transactions are nested, to ensure that the correct database connection is identified.

Contains

sqld:column, SQL driver datatype elements

Attributes

AttributeDescriptionTypeDefaultOptionsUse
dataSourceName of a valid datasource as specified in the mcs-config.xml file.xs:stringnone  required 
sqlSpecifies the query that is to be applied to the datasource.xs:stringnone  required 
transactionIdentifies the transaction to which the statement belongs. Required in nested transactions.xs:stringnone  optional 

Example

<sqld:query
   datasource="PressReleases" 
   sql="select TITLE,ABSTRACT 
      from PRESS_RELEASES 
      where STORY = ?">
   <sqld:string value="%{request:getParameter('STORY')}"/>
   <sqld:column name="ABSTRACT">
      <mdc:parseAsXML
         defaultNsPrefix="pr"
         defaultNsURI="http://mycorp.com/schemas/pr"/> 
   </sqld:column>
    ...
</sqld:query>
