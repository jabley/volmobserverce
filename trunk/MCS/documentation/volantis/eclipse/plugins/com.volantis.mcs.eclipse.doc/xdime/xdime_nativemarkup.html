nativemarkup

Purpose

Used to define device specific markup. The content model is xs:any.

If the protocol is not WML, either the expr attribute is required, or the nativemarkup elements must be within the scope of a select element.

If an invalid location is specified in the markup, a warning message is logged and the page will be generated without the data specified for that location.

Note: Some targetLocation attribute values are WML-specific.

Contained by

canvas

Attributes

AttributeDescriptionTypeDefaultOptionsUse
exprAn MCS expressionxs:stringnone  optional 
paneThe name of the pane if targetLocation is set to 'pane'xs:stringnone  optional 
targetLocationSpecifies where the markup is to be placed in the page. It can be a literal or a text component.xs:stringnone wml.deck.template, wml.card.timer, wml.card.onevent, wml.card.beforebody, here, pane, html.head, vdxml.head required 

Example

<canvas layoutName="/wml_events.mlyt"
   cacheScope="none"
   id="card1"
   type="inclusion"
   overlay="true"
   title="Abbreviated">
  <nativemarkup targetLocation="wml.card.onevent">
     <onevent type="onenterforward">
        <refresh>
           <setvar name="var" value="Aldrin, Buzz"/>
           <setvar name="varEsc" value="Dragon%2C+Snap"/>
        </refresh>
     </onevent>
   </nativemarkup>;
</canvas>
