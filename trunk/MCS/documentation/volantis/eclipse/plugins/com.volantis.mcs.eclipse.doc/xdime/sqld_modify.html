sqld:modify

Purpose

Container for SQL modify statements that can insert, update, or delete entries in database tables.

Note: The transaction attribute binds the SQL statement to the transaction. Although it is optional the transaction attribute should always be used when transactions are nested, to ensure that the correct database connection is identified.

Contains

SQL driver datatype elements

Attributes

AttributeDescriptionTypeDefaultOptionsUse
dataSourceName of a valid datasource as specified in the mcs-config.xml file.xs:stringnone  required 
sqlSpecifies the modification that is to be applied to the datasource.xs:stringnone  required 
transactionIdentifies the transaction to which the statement belongs. Required in nested transactions.xs:stringnone  optional 

Example

<sqld:modify
   dataSource="address"
   sql="insert into ADDRESS (FORENAME, SURNAME, ADDRESS) values (?, ?, ?)">
   <sqld:string value="%{request:getParameterValue('forename')}"/>
   <sqld:string value="%{request:getParameterValue('surname')}"/>
   <sqld:string value="%{request:getParameterValue('address')}"/>
</sqld:modify>
