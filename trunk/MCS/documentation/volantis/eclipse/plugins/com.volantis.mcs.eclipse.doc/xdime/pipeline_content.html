pipeline:content

Purpose

Allows you to specify mixed content. Pipeline markup within the element will cause a streaming error.

The element is parsed as an XML document fragment, which means that it can contain multiple root elements and text at the top level. Its body can contain other elements, but they will only have an effect on the markup if they write to the element's body content; if they do so, the result must be well-formed XML.

Example

<pipeline:try> 
   <pipeline:preferred> 
      <pipeline:transform href="title.xsl" compilable="true">
         <urid:fetch href="title.xml" />
      </pipeline:transform>  
  </pipeline:preferred> 
  <pipeline:alternative> 
      <pipeline:content>
         <p>There was a problem retrieving the information requested.</p>
      </pipeline:content> 
  </pipeline:alternative> 
</pipeline:try>
