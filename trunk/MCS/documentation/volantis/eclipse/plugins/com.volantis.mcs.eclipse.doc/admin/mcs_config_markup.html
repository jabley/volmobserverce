Defining markup plugins

You can define markup plugins in the mcs_plugins section of the mcs-config.xml. Markup plugins are custom Java classes that use the com.volantis.mcs.integration.MarkupPlugin interface. Authors can invoke a class from XDIME using the vtim:invoke element.

To define a plugin you use the attributes of the markup_plugin element. The name and class attributes name the plugin and the class that will be called. The scope attribute specifies whether the plugin will be initialized and terminated when the application, session, or canvas generation starts and ends.

The arguments to initialize the plugin are specified as name-value pairs in the argument element.

<mcs-plugins>
   <markup-plugin name="YOUR-PLUGIN-NAME"
      class="your.class.Name"
      scope="application">
      <initialize>
         <argument name="NAME1" value="VALUE1"/>
         <argument name="NAME2" value="VALUE2"/>
      </initialize>
   </markup-plugin>
</mcs-plugins>
