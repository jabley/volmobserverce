Configuring a publishing server

MCS can host multiple Secure Publishing Server (SPS) instances using a Secure Connection Server (SCS) proxy.

When an SPS retrieves a request from the Secure Connection Server (SCS) it will rewrite the URL specified in the GET or POST request to a URL of a local web server. The mapping is configured in the sps-config.xml file, which defines the prefix for the local URL and the URL specified in the response to the authentication request.

The secure-connection element defines the details for connection to the SCS. The host and port attributes specify the location of the listening port of the SCS. The userid and password attributes are sent to SCS to authenticate this connection so that the client URL can be retrieved, and the secure connection maintained. The web-server element specifies the location of the local web server that the SCS is proxy for. The base-url attribute is used to substitute against the client URL returned from the SCS authentication. The log4j element points to a log4j.xml file defining the logging appenders.

<sps-config>
   <secure-connection-configuration
      host="scs.myhost.com"
      port="1067"
      userid="newsproject"
      password="secret"/>
   <web-server base-url="http://newsserver:8080/resources"/>
   <log4j
      xml-configuration-file="/[path_to_sps]/config/scs-log4j.xml"/>
</sps-config>
