Asset attributes

In MCS editors you can edit the attributes of the assets used in components. The attributes for an asset depend on its type. Some attributes such as value are common to all assets. Others are more specific.

Attribute definitions

This table gives the meaning of the different asset attributes. You'll see these values in different combinations in MCS component editors, depending on the type of asset.

Attribute Description
Value URL or file name of an asset. If you use a filename, MCS treats it as relative to the Prefix URL given in the asset group. Alternatively, you can specify an absolute URL. If you specify a relative URL, MCS creates an absolute URL based on the name of the server that received the request.
Value type Type of value (URL or literal) for text assets
Type Type of text asset (Plain, Volantis Markup, XML Markup, Voice XML Help, Voice XML Prompt, Voice XML Error, Voice XML Nuance Grammar or Form Validator)
Asset group name Name of the asset group an asset belongs to
Device name Name of the device or device group to which an asset is targeted. Only valid if the asset is device specific.
Encoding Encoding of the audio clip (Adcpm32, GSM, Basic, Midi, MP3, Windows Media Audio, Real Audio), or image (BMP, GIF, JPEG, PJPEG, PNG and WBMP)
X Title Title for the X axis of a chart
Y Title Title for the Y axis of a chart
X Interval Interval for data points on the X axis of a chart
Y Interval Interval for data points on the Y axis of a chart
Width hint Suggested percentage of the pane to be used by an image
Rendering
 Rendering required (color or monochrome)
Height
 Height in pixels of the image to be displayed
Width
 Width in pixels of the image to be displayed
Pixel depth Depth in pixels of the image to be displayed
Programming language
 Programming language of a script
Mime type
 MIME type associated with the scripting language. For example, text/javascript
Character set Character set in which a script is written
