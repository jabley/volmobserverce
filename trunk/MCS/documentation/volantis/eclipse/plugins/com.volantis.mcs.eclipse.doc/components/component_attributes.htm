Component attributes

The table shows the relationship between component attributes and the components they can refer to. It lists component attribute names in two groups on the left; fallback attributes, then specific attributes that apply only to certain types. The components with these attributes (refer to these components) are tabulated across the top. Names are shortened for clarity.

The shaded background shows the relationships. For example Dynamic Visual components can have up to four fall backs. At the other extreme all components can have a Text component fall back

The only component type without any attributes is the Script component.
