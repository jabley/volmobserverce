Form properties

Purpose

Specifies the appearance and positioning of input and other objects used in forms.

ControlDescriptionOptions
Action Style

How form action controls should be displayed. The default is usually a button. For WML devices a link, which uses the anchor tag, can be specified instead of a button.

default, image, link 
Action Image

Image component or URI to be used when a form action has a Action Style of image. Browse opens the Image Component Selection dialog.

none, [image_component], URI 
Indicating Image

Image component or URI to be used next to, or on items such as an anchor

none, [image_component] URI 
Columns

Width in characters of a text input control

 
Rows

Number of rows in a text input control

 
Selection List Layout

Order of the captions and controls in the display if both occupy the same pane

caption-first, control-first 
Selection List Style

Type of the selection list input controls

controls, default, circular-list, menu 
Selection List Trigger

Specifies the event that triggers display of a selection list

normal, focus, select 
Focus

Whether or not to accept focus

accept, ignore 
HTTP Method Hint

The HTTP method to use

get, post 
Break after

Applies to xf:group and widget:wizard elements. The default value of 'never' means that the group does not end a form fragment. If set to 'always' then the group ends a form fragment, and another fragment begins immediately.

always, never 
Input format

Defines the pattern to be used for validation
