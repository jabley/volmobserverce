Displaying a feed

If a ticker:feed element is inserted as a child of the Carousel or Ticker Tape widgets, it makes these widgets show feed obtained by the Feed Poller from BuzzCast Server. Clicking on an item in Carousel or Ticker Tape widget will show details of this item in the associated Item Display widget.

XDIME 2 elements

Feed Carousel widget:

<widget:carousel>
   <ticker:feed channel="ImportantNews" item-display="myItemDisplay"/>
</widget:carousel>

Feed Ticker Tape widget:

<widget:ticker-tape>
   <ticker:feed channel="ImportantNews" item-display="myItemDisplay"/>
</widget:ticker-tape>

You may configure a template for a single item displayed in a carousel or ticker tape. It can contain such item components as title, icon and plain description.

<widget:ticker-tape>
   <ticker:feed>
      <ticker:item-icon/>
      <ticker:item-title/>: <ticker:item-plain-description/>
   </ticker:feed>
</widget:ticker-tape>

Supported style properties

All style properties specified for Carousel and Ticker Tape widgets can be applied. In addition, the mcs-item pseudo-element combined with the mcs-between pseudo-element can be used to insert separator between items. The content property provides a content for the separator. 

ticker:feed::mcs-item::mcs-between {content: " *** ";}
Note: Separator currently can be applied only if the ticker:feed element is placed within the Ticker Tape widget. It has no effect if used within the Carousel widget.

Non-client fallback

If the BuzzCast Client is not supported on the device, the widget displays its initial content only.

If the enclosing widget is not supported on the device, the widget's initial content is displayed as specified by the fallback behavior.

JavaScript API

The Carousel widget public API is extended with nextPage() and previousPage() methods allowing the displayed carousel items to be moved forwards or backwards manually. Supported JavaScript methods for the Ticker Tape widget remain the same. Please see the JavaScript support for details.
