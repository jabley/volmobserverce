Glossary

This glossary defines some common terms used in MCS and web applications. It shows MCS icons for objects like components and layout formats, and contains links both to other terms, and to further information.

Asset
An asset is a file containing partial content for a web page, including images, audio, scripts and text.
Asset group
The URL of a group of assets on a server or a device.
Audio
MCS component containing references to audio assets.
Canvas
A layout area where you can position formatting objects and associate them with elements in your web content. On a PC, MCS delivers a canvas as a single web page. On smaller devices MCS sends complex canvasses as sets of pages.
Cascading Style Sheet (CSS)
A mechanism for adding style properties such as fonts, colors, spacing to Web documents. See the Cascading Style Sheet specification for further details.
Component
Component names identify sets of assets, such as images or dynamic visuals, which are available for a range of devices. MCS chooses an appropriate asset for the requesting device.
Composite Capabilities/Preferences Profiles (CC/PP)
A CC/PP profile is a description of device capabilities and user preferences using RDF/XML syntax.
Content
Basic information delivered to users by a web site. It may be static information, or dynamic data retrieved in response to a query.
Chart
An MCS component type that displays data contained in XDIME content.
Device
A hardware device such as a PC or a mobile phone. MCS stores device attributes in a repository, and uses this information to match content presentation to the device.
Device policy category
Categories of device policies such as audio, identification and protocol that group device characteristics together.
Device repository
A compressed XML file defining a set of devices. MCS comes with a pre-defined repository containing a large number of devices that is frequently updated.
Dissecting pane
A pane that MCS can automatically split up and send to a small-screen device as one or more linked shards.
Dynamic visual image
An MCS component type for defining assets such as video and animation.
Fallback
A resource that can be used if MCS fails to match the capability of device with a preferred resource, such as a monochrome image replacing a high-resolution color image. Fallback relationships are represented in MCS as component attributes. Device layouts and themes also belong to fallback hierarchy.
Form fragment
A layout format used to divide a form into fragments on small-screen devices.
Format
A layout object, such as a grid, pane or form, that will contain either other formats or specified content on a web page.
Fragment
A layout format used to divide the content of a canvas into fragments that can be displayed on small screens such as PDAs and mobile phones.
Generic asset
An asset such as an image that will display successfully on a wide range of devices without requiring complex device analysis.
Grid
A layout format used to position content in a row/column structure in a web page.
Image
An MCS component type containing references to image assets.
Iterator pane
A pane that will generate row or column cells to display a data series. The display can be spatial or time based.
Identification pattern
MCS uses identification patterns at runtime to identify an unknown device. The patterns are regular expressions that are matched against the device headers.
Layout
Layouts specify the physical placement of content on the display of a user's device.
Link
MCS component type containing references to hyperlink assets.
Multi-channel Server (MCS)
MCS enables the integration of input from a variety of sources, and delivery of output to multiple target devices.
Montage
A montage is a collection of canvasses that can be rendered together on some kinds of device. On smaller devices, a is rendered as a set of individual pages.
Pane
An area in a canvas layout for displaying content.
Policy
Information that defines MCS responds to particular devices. There are three types of policy within MCS covering devices, layouts, and themes.
Portal
Web site that combines material from a variety of sources into a single presentation.
Portlet
The content that occupies a region in a portal.
Property
A single value in a style sheet rule, for example a font-weight.
Project
A project is a set of resources that you define for a particular web application. Resources that make up a project could include XML files, assets, layout, theme, component and device policies.
Replica
A replica format can use content from another format in another part of a layout.
Region
A region format is one of a set of areas in a portal.
Rollover image
MCS component type that contains references to the images assets that highlight a rollover image.(normal, over).
Script
MCS component type that contains references scripts that are contained in a web page.
Segment
A segment is an area in a montage layout that specifies which canvas or montage will be displayed in it.
Selector
One or more element names, class names or identifiers that are used to associate style properties with some content.
Shard
A section of content with navigation links that MCS creates automatically when a dissecting pane generates page content for a device with limited page capacity.
Spatial iterator
An iterator pane that displays a data set in a tabular layout.
Stem
Part of an identifier (the pane attribute) used to map XDIME content on to a cell in an iterated row or column. It is inherited from the name of the iterator pane that generates the cell. The other part of the identifier is the tail.
Style rule
A set of style properties for a selector applied to a device.
Tail
Part of an identifier (the pane attribute) used to map XDIME content on to a cell in an iterated row or column. It is a zero-based numeric index that is appended to the name or stem inherited from the iterator pane in the form name.index.
Targeted asset
Assets that are associated with a specific device.
Temporal iterator
An iterator pane that displays a data set over time.
Text
MCS component type that contains references to the text assets for a web page.
Theme
All the information that governs the stylistic aspects of a web site. So, for example, a corporate brand would be a theme, composed of styles on each device.
Untargeted asset
Assets that are not associated with a specific device, so finding the most appropriate asset can be quite complex.
Uniform Resource Locator (URL)
A URI that identifies a resource using a network location rather than identifying the resource by name or some other attribute(s).
Uniform Resource Identifier (URI)
The name of resource. The resource does not need to have a network location, so a URI can represent a physical or abstract entity.
World Wide Web Consortium (W3C)
W3C develops interoperable technologies (specifications, guidelines, software, and tools) for the Web.
XML Device Independent Mark-up Extensions (XDIME)
An XML vocabulary based on HTML and WML that is used to describe MCS content.
