Validation

When MCS validates the content of work you have done, it reports any problems. What you see depends on the situation.

Policy editor errors

In the Alerts and Action Items section of a policy editor, the error information is updated each time you make a change. For example when you add a new asset to a component, MCS validates the change and reports on any required attributes that are missing. As you enter valid attribute data the list changes until there are no errors showing.

If there are any current errors, you'll see a link to the relevant items in the Task view, for example, 'There are 2 errors 0, warnings and 3 tasks associated with this theme'. You can click on the link to highlight the items in the Task list. If there are no errors, you'll see 'No alerts at this time'.

Errors in the Task view

The same problems are also included in the workbench Task view. If the Task view is closed when MCS first reports an error, MCS opens it the Task view to show the problem.

The task list contains all the outstanding items that need your attention. When you Go To an MCS resource on the Task list, MCS brings the resource tab to the front (or opens the editor if it is closed), and highlights the control where the problem is.

Errors in dialogs

Errors in dialogs are shown by text in the body of the dialog.

Errors in lists

Errors in asset lists are indicated by an asset icon with an error decoration.
