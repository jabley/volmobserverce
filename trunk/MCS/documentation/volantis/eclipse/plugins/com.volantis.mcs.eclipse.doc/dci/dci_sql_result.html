Query and modification results

There is a separate namespace with the prefix sqlr: for the results of queries and modifications. In the case of queries the data is returned in a row/column format that makes it easy to transform the structure into XDIME tables.

The container for the result is the sqlr:result that contains multiple sqlr:row elements.

The parameter element names used here for data types are identical to those in the sqld: namespace, with the addition of the exml element that can return markup from any namespace. In all cases the column attribute specifies the name of the column from which the value is returned.

<sqlr:result>
   <sqlr:row>
      <sqlr:string column="FORENAME">Ian</sqlr:string>
      <sqlr:string column="SURNAME">Grant</sqlr:string>
      <sqlr:string column="ADDRESS">ig@somewhere.net</sqlr:string>
   </sqlr:row>
   <sqlr:row>
      <sqlr:string column="FORENAME">Alan</sqlr:string>
      <sqlr:string column="SURNAME">Dawes</sqlr:string>
      <sqlr:string column="ADDRESS">al.dawes@mycorp.com</sqlr:string>
   </sqlr:row>
   ...
</sqlr:result>

The result of a modification is a count in the sqlr:updateCount element of the number of rows that have been updated or modified.

<sqlr:result>
    <sqlr:updateCount>4</sqlr:updateCount>
</sqlr:result>
