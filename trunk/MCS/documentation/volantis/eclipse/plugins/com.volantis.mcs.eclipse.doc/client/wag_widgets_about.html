About widgets

Framework Client provides developers with a simple, standards-compliant authoring environment. Control over widget behavior and transitions is available as either policy theme properties or as attributes on markup. In this section authors can find all the information needed to create graphically appealing client applications with more client-side interactivity than standard mobile browser applications are able to provide.

Tip: Some widgets support transitions, such as sliding, fade in, etc. See the About transitions section for details.

Fallback styling

It was mentioned that if the device does not support the Framework Client, the server-side processing of the XDIME 2 widget provides an alternative static rendering that assures the basic function for which the widget was designed. Actions and responses

Many widgets are active, in that they are able to load or refresh their content or status. These actions are implemented using Asynchronous JavaScript and XML (AJAX).

You use the XDIME response vocabulary to provide data for widgets. You should always use this vocabulary; any other approach might result in unexpected behavior. Refer to Widget response structure for more information.

You can allow users to suspend content polling through a provided JavaScript API, but be note that it affects only automatic interval-based polling. Polling triggered by the user or JavaScript API is not affected. See JavaScript support for details.
