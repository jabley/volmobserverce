Device policy categories

In the Policies section of the Device Repository view you can examine a list containing device attributes. By clicking on the category column head you can sort the attributes into the categories in the table.

The same categories are displayed in the Policy page of the Device Repository editor.

This table describes the categories and gives some examples of policy names.

Category Description Example policy names
Audio The device's ability to play audio clips GSM, MIDI, MP3
Browser Basic attributes of the browser WML versionl, Browser name
Custom A category for implementation-specific policies; not part of the standard device repository  
Dynamic Visual The device's ability to show dynamic visuals Animated GIF, Real Video
Ergonomics Usability factors Weight, Viewing distance
Identification Attributes that identify the device Manufacturer, Model number
Image The device's ability to display images WBMP, JPEG
Input Capabilities relating to text or drawing input, and other inputs such as barcode scanners and video cameras Keyboard type, Barcode reader
Location Device-based Location detection attributes Map datum, Positioning protocol
Message Message-related information MHTML, MMS, SMS
Miscellaneous Miscellaneous attributes  
Network Attributes covering network aspects Assumed bandwidth, GPRS class
Output Physical attributes of the primary visual display or other output mechanisms Chroma bleed, Pixel depth
Protocol Attributes used to control protocol-specific behavior in MCS, at the device level. WML emulation, Stylesheet location
Rules The short names of the conversion rules to be used. ICS rule names
Security Capabilities relating to the information security of the device or the protocols that it supports Secure sockets version, Password hidden
System Internal hardware attributes such as processor speed, memory capacity etc. Processor clock rate, Software version
