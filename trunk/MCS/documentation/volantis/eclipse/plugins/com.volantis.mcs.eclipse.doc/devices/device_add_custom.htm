Adding a custom policy

On the Structure page of the Device Repository editor you can provide the definitions for names, type, possible values, and other information that combine to make a device repository policy.

  1. In the Device tree select Custom as a fallback. MCS enables the New button.
  2. Click New to start the New Policy wizard
  3. Enter a Short name . Policy names must be unique in a repository, so if you use the name of an existing device you'll see a warning.
  4. Choose a combination of Composition and Type for the policy. The options in Type will change depending on the value in Composition.
  5. Click Finish to close the wizard. MCS displays the values you have entered in the Policy definition section. Short name and Composition are read only.
  6. In the Policy Definition section add a display Name. MCS also displays this text in the device tree.
  7. Enter a Description that will appear as a tool-tip for the policy controls
  8. Enter or Add the value(s) that you want for the category, If it is a selection you can use the Up or Down buttons to order your list.
  9. Define policy value(s) in the Master Value section. These may be any value or values specified for the new category.
  10. Add a UAProf name

MCS displays the policy control you have defined the next time you go to the Policies page and select the Custom category.

Note: The user interface guidelines for Eclipse recommend that you should use sentence style (initial letter only) capitalization for display names.

To remove a policy from the Custom category, select it and click Delete.
